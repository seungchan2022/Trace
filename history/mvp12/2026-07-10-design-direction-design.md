# MVP12 design-direction — Trace 디자인 시스템 (2026-07-10)

MVP12 `design-direction` 마일스톤의 산출물. 사용자가 Claude Design으로 만든 목업
**"Trace 경로 짜기 v2"**를 기준으로 디자인 시스템을 추출·문서화하고, 목업이 다루지 않는
화면(코스 저장/목록/왕복/redo)을 같은 언어로 확장한다. 이 문서는 `design-apply` 사이클의
단일 기준이다. **코드 변경 없음** — 적용은 Swift 6 사이클(`swift6-migration`) 완료 후.

## 0. 출처와 확정 결정

- **원본**: Claude Design 프로젝트 `a8064d66-dce6-40aa-a352-5232d49415c5`,
  파일 `Trace 경로 짜기 v2.dc.html` (2026-07-10 스냅샷 기준. `.dc.html`은 프로젝트 내
  `support.js`/`ios-frame.jsx` 없이는 렌더되지 않아 저장소에 복사하지 않는다 —
  Claude Code에서는 DesignSync `get_file`로 재조회 가능, 수치는 이 문서에 전부 추출됨)
- **콘셉트**: 하이브리드 테마 — 다크 = "Cyber Performance"(짙은 네이비 + 네온 민트 + 글로우),
  라이트 = "Glassmorphism"(밝은 지도 + 반투명 유리 + 딥틸)
- **확정 결정 (2026-07-10 인터뷰, 사용자)**:
  1. **변형 노브**: 목업 기본값 조합 확정 — accentFamily `mint` · routeStyle `harmonic`(구간별 색)
     · routeEnergy `balanced` · mapDensity `standard` · cornerStyle `default` · numberStyle
     `geometric` · sheetSurface `auto`(다크 솔리드/라이트 유리)
  2. **폰트**: 번들 없음, 네이티브 SF — UI는 SF Pro(시스템 기본), 거리·숫자는
     SF Rounded + tabular(`.monospacedDigit()`). 한글은 시스템 폰트(Apple SD Gothic Neo) 그대로.
  3. **테마**: 시스템 다크모드 연동. 라이트/다크 둘 다 구현하되 선택은 iOS 시스템 설정이 하고,
     **인앱 테마 토글은 만들지 않는다**(목업의 해/달 버튼 제거 — 그 자리는 §3.1 코스 목록 버튼).

## 1. 디자인 토큰

### 1.1 컬러 — 시맨틱 토큰 (라이트/다크 쌍)

Asset Catalog 컬러(다크 변형 포함) 또는 `Color` 확장으로 구현. 이름은 목업 변수를 따른다.

| 토큰 | 라이트 (Glass) | 다크 (Cyber) | 용도 |
|---|---|---|---|
| `ink` | `#0C1A22` | `#EAF2FB` | 주 텍스트·아이콘 |
| `ink2` | `#5B6B78` | `#65778E` | 보조 텍스트 |
| `surface` | `rgba(255,255,255,.74)` | `#0A111E` | 바텀시트 배경 |
| `surface2` | `rgba(6,182,164,.09)` | `#141F2F` | 칩·선택 행 배경 |
| `border` | `rgba(12,26,34,.09)` | `rgba(255,255,255,.07)` | 구분선·행 테두리 |
| `glass` | `rgba(255,255,255,.66)` | `rgba(10,17,30,.72)` | 유리 크롬(탑바·FAB) 배경 |
| `glassBrd` | `rgba(255,255,255,.7)` | `rgba(0,229,176,.18)` | 유리 크롬 테두리 (다크는 민트 틴트) |
| `accent` | `#06B6A4` (딥틸) | `#00E5B0` (네온 민트) | 브랜드·활성 상태·출발 핀 |
| `accentInk` | `#FFFFFF` | `#04120E` | accent 위 텍스트 |
| `danger` | `#F43F5E` | `#FF3B6B` | 에러·도착 핀·삭제 |
| `locBlue` | `#0284C7` | `#38BDF8` | 현재 위치 |
| `grabber` | `rgba(12,26,34,.2)` | `rgba(0,229,176,.4)` | 시트 그래버 |
| `markerFill` | `#FFFFFF` | `#0A1420` | 지도 마커 원 배경 |

- **구간 색 팔레트 `seg[6]`** (colorKey % 6로 순환, 기존 `segmentColorKeys` 로직 유지):
  - 라이트: `#06B6A4` `#0E9488` `#0284C7` `#0891B2` `#059669` `#0D9488`
  - 다크: `#00E5B0` `#2FE0C6` `#38BDF8` `#5EEAD4` `#22D3EE` `#34D399`
- **그림자**: 라이트 `y14 blur40 rgba(20,60,70,.16)` / 다크 `y14 blur44 rgba(0,0,0,.6)` → SwiftUI `.shadow` 근사
- 목업의 지도 바탕색(land/park/water 등)은 가짜 SVG 지도 전용 — 실제 앱은 MapKit 타일이므로 토큰이 아니다.
  다크 지도는 시스템 다크모드 시 MapKit이 자동 처리.

### 1.2 타이포그래피

| 역할 | 스펙 (SwiftUI) |
|---|---|
| 거리 헤드라인 | 44pt bold `.rounded` + `.monospacedDigit()`, 단위 "km"는 17pt semibold `.rounded` `ink2` |
| 구간 행 거리 | 16pt semibold `.rounded` + tabular |
| 구간 행 제목 | 15pt semibold / 부제(타입 라벨) 12.5pt medium `ink2` |
| 섹션 라벨 ("구간 N") | 12pt semibold, tracking 0.6, `ink2` |
| 상태 칩·힌트 필 | 13~13.5pt semibold(에러 칩은 bold) |
| 서브타이틀 | 13.5pt medium `ink2` |

### 1.3 모양·간격

- 코너 반경: 크롬(버튼·FAB·세그먼트) **15pt** · 바텀시트 상단 **26pt** · 리스트 행 **15pt** · 칩/필 **캡슐**
- 크기: FAB **44×44** · 탑바 버튼/뱃지 **42×42** · 화면 좌우 여백 **14pt** · 시트 내부 패딩 **20pt**
- 유리 재질: blur 18~26 + saturation → `.ultraThinMaterial`(크롬)/`.regularMaterial`(라이트 시트) +
  1px `glassBrd` 테두리. 다크 시트는 유리 아님(솔리드 `surface`, sheetSurface=auto).

### 1.4 모션

- 핀 등장 pop: scale 0.5→1 + fade, 0.28s ease-out → `.spring` 근사
- 현재 위치 pulse: 2.6s 반복 확산
- 시트 높이 전환: 0.32s `cubic-bezier(.22,1,.36,1)` → spring
- 상태 칩/힌트 전환: 0.2s fade
- 그리는 중 점선 행진(dash 0.6s 반복): MapKit 오버레이 구현 난이도 있음 → **P2** (§5)

## 2. 플래너 화면 컴포넌트 (목업 커버 영역)

- **탑바** (top 52, 좌우 14): 좌 — 로고 뱃지(42pt, `accent` 배경, 러너+경로 글리프; 추후 앱 아이콘 후보).
  중앙 — 탭/그리기 세그먼트(유리 필, 활성 탭은 `accent` 배경 + `accentInk` 텍스트).
  우 — ~~테마 토글~~ → **코스 목록 버튼** (§3.1).
- **힌트 필** (top 108 중앙, 캡슐): 그리기 모드 안내("손으로 경로를 그려보세요") 및 에러
  (배경 `danger`, 흰 텍스트, **2.6초 자동 해제**). 현재 앱의 에러 메시지 표시를 이 형태로 통일.
- **FAB 스택** (우측, 시트 상단에서 16pt 위, 유리 44pt): 편집 중일 때 undo·**redo(확장)**·clear,
  항상 내 위치. 시트 확장 시 숨김(fade+slide).
- **바텀시트**: 그래버(38×5 캡슐) → 거리 헤드라인 + 서브타이틀(상태별 문구, §2.1) + 우측 상태 칩.
  확장 시 구간 리스트. *현재 앱의 "지도 위 고정 구간 패널(최대 40%)"을 시트 UI로 재구성하는 것* —
  드래그 확장은 P2, 탭 토글 확장은 P1 (§5).
- **구간 리스트 행**: 색 바(10×34, r6, `seg` 색) + "구간 N"/타입 라벨("지점 연결"·"직접 그리기") +
  거리. 선택 시 `accent` 테두리 + `surface2` 배경 (+ 지도 halo는 P2).
- **상태 칩** (시트 헤더 우측): 계산 중(스피너+"계산 중") / 에러("다시 시도", `danger`) /
  출발 지정됨(민트 점) / 경로 있음("구간 N" + 확장 셰브론).
- **지도 위 표현**: 구간별 `seg` 색 폴리라인 + 케이싱(라이트 흰색 .6 / 다크 검정 .35 — 다크 글로우는 P2),
  선 폭 5.5(케이싱 9.5). 출발 핀 = `accent` 원(r14.5) + 러너 글리프, 도착 핀 = `danger` 원 + 깃발
  글리프, 흰 테두리 + 그림자. 현재 위치 = `locBlue` 펄스. km 마커 뱃지는 P2.

### 2.1 상태별 서브타이틀 문구 (목업 확정 문구)

| 상태 | 문구 |
|---|---|
| 빈 상태 | "지도를 탭해 출발지를 선택하세요" |
| 출발 지정됨 | "이제 도착지를 탭하세요" |
| 그리기 모드 | "지도에 손으로 경로를 그려보세요" |
| 계산 중 | "경로를 계산하고 있어요" |
| 경로 있음 | "도보 기준 · 탭해서 이어 그리기" |
| 에러 | "도로에 더 가까운 지점을 눌러보세요" (필: "도보 경로를 찾을 수 없습니다" 등 원인별) |

## 3. 목업 미커버 화면 확장 (같은 언어로 신규 설계)

목업에 없는 MVP11 기능들. 아래는 목업의 토큰·형태 규칙을 그대로 적용한 확장안이며,
세부 배치는 design-apply에서 실물 화면 보며 조정 가능(UI 세부는 확인 없이 진행).

### 3.1 코스 목록 진입점 + 목록 sheet

- 진입점: **탑바 우측 42pt 유리 버튼** (테마 토글 자리), 폴더/북마크 글리프.
- 목록: 표준 `.sheet` — 상단 그래버 + "저장한 코스" 타이틀(15 semibold). 행 = 좌측 `seg[0]` 색 바
  + 코스 이름(15 semibold) + 부제 "거리 · 저장일"(12.5 `ink2`), 행 코너 15pt, 스와이프 삭제(`danger`),
  탭 = 불러오기. 빈 상태 문구 "저장한 코스가 없어요"(13.5 `ink2`).

### 3.2 코스 저장

- 진입점: **시트 헤더의 저장 버튼** — 경로가 있을 때 상태 칩 왼쪽에 `accent` 캡슐 버튼
  ("저장", `accentInk` 텍스트, 13 bold). 좁으면 아이콘(북마크+)만.
- 이름 입력은 **현재의 시스템 알럿 유지** (커스텀 다이얼로그는 P2— 킥오프에서 기각 가능).
  ⚠️ 재구성 시 MVP11 버그픽스 **`.ignoresSafeArea(.keyboard)` body 최상위 적용을 반드시 보존**
  (`docs/solutions/design-patterns/swiftui-keyboard-avoidance-shrinks-representable.md`).

### 3.3 왕복 액션

- **구간 왕복**(자유 끝 구간 전용, MVP11 시맨틱스 유지): 해당 구간 행 trailing에 왕복 아이콘 버튼
  (화살표 U턴 글리프, 42pt 히트 영역).
- **전체 왕복**: 구간 리스트 헤더 우측("탭해서 지도에서 보기" 자리)에 텍스트 버튼 "전체 왕복"(12 semibold `accent`).

### 3.4 redo

- FAB 스택에서 undo 아래 redo 버튼(앞으로 화살표 글리프). 불가 시 40% 투명도로 비활성 표시.

## 4. SwiftUI 이식 가이드

- **토큰 구현 위치(제안)**: `Trace/DesignSystem/` 신설 — `Tokens.swift`(컬러·수치),
  공용 컴포넌트(`GlassButton`, `StatusChip`, `HintPill` 등). Pages와 별도의 presentation 공용
  레이어이며 추후 모듈 분리 대상. *새 최상위 폴더라 design-apply 플랜에서 아키텍처 확인 후 확정.*
- 컬러는 Asset Catalog(Any/Dark 변형)로 등록해 시스템 테마 연동을 공짜로 얻는다.
- 지도: MapKit 타일·다크 전환은 시스템에 위임. `mapDensity=standard` = 현재 POI 기본값 유지.
  폴리라인 케이싱은 같은 좌표로 굵은 선을 아래 겹치는 2-pass 렌더링.
- 도메인·persistence 계층은 **변경 없음** — design-apply는 Pages/UIComponent(+표시용 ViewModel
  상태 추가) 범위. 기존 attach·왕복·저장 시맨틱스에 손대지 않는다.

## 5. design-apply 범위 게이트 (P1 / P2)

- **P1 (디자인 적용 — design-apply 기본 범위)**: 토큰·타이포 전면 적용, 탑바/세그먼트/FAB/시트
  외형 재구성(확장은 탭 토글), 구간 리스트 스타일, 핀·폴리라인 색/케이싱, 상태 칩·힌트 필·문구,
  저장/목록/왕복/redo 재배치(§3).
- **P2 (기능 추가 성격 — design-apply 킥오프에서 포함 여부 결정)**: 시트 드래그 리사이즈,
  구간 선택 시 지도 halo 하이라이트, km 마커 뱃지, 그리는 중 점선 행진 애니메이션,
  다크 폴리라인 글로우, 커스텀 저장 다이얼로그.

## 6. 검증 기준 (design-apply에서)

- 라이트/다크 각각에서 §2 컴포넌트가 토큰 표와 일치(스크린샷 대조).
- 기존 기능 회귀 없음: 탭/그리기, undo/redo/clear, 저장/불러오기/삭제, 구간·전체 왕복, 에러 안내.
- 저장 알럿 중 지도 줌아웃 버그 재발 없음(키보드 회피 보존).
- 실기기 QA 체크리스트는 MVP12 완료 시 별도 작성(`docs/qa/`).
