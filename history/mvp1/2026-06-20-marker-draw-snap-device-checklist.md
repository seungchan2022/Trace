# 마커 그리기 + 스냅 실기기 체크리스트 (2026-06-20)

브랜치: `feature/marker-draw-snap`. 정책: `docs/agent-rules/testing.md`의 Real-Device Verification.
이미 검증됨(자동/시뮬): 단위 25개 통과, 시뮬레이터(서울 강남)에서 그리기→스냅→거리(예: 1.52km)·safe area·되돌리기/초기화 시각 확인.
아래는 **실기기에서만 닫히는 빈틈**(GPS/권한/터치 감/한국 실측/의도 일치). **위에서 아래로 순서대로** 진행하세요.

## 1. 빌드 · 설치
- [ ] 기기 USB 연결 → Xcode에서 타깃 기기 선택 → Run 성공 (서명 Automatic, 팀 `D22ZM93S77` 설정됨)
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰: 설정 > 일반 > VPN 및 기기 관리 > 개발자 앱 신뢰

## 2. 첫 실행 · 위치 권한
- [ ] 위치 권한 팝업이 뜨고 문구가 "달릴 코스를 현재 위치에서 시작하기 위해…" 로 보임
- [ ] 허용 → 지도가 **실제 내 현재 위치**로 줌인됨

## 3. 그리기 시작
- [ ] "그리기" 탭 → 버튼이 "그리기 종료"로 바뀜
- [ ] 그리기 모드에서 드래그하는 동안 **지도가 움직이지 않음**(그리기로 동작)

## 4. 그려서 스냅 확인 (한국 실측 — 핵심)
- [ ] 손가락으로 **구불구불한 경로**를 그림 → 손 떼면 파란 경로가 **실제 보행로에 붙고** 하단에 거리(km) 표시
- [ ] **스냅 품질**: 건물 통과·엉뚱한 우회 없이 도로 격자를 따라가는가
- [ ] 그린 모양과 스냅 경로가 **내가 의도한 길과 대체로 일치**하는가
- [ ] 그리기 반응 속도가 답답하지 않음(실기기 체감)

## 5. 이어 그리기 · 되돌리기
- [ ] 한 번 더 이어 그림 → 경로가 누적되어 재계산됨
- [ ] "되돌리기" → 마지막 구간만 사라지고 경로 재계산

## 6. 초기화
- [ ] "초기화" → 그린 경로·거리 모두 사라짐

## 7. 모드 복귀 (탭 방식 공존)
- [ ] "그리기 종료" → 지도 이동(pan) 정상으로 돌아옴
- [ ] 지도를 탭 → 기존 출발/도착(`포인트`) 마커 동작

## 8. 엣지 · 견고성
- [ ] **빠르게 연속으로** 여러 번 그리기 → 경로가 어긋나거나 옛 경로가 되살아나지 않음(레이스 가드)
- [ ] 매우 길고 복잡한 경로 → 느려지거나 "경로를 계산할 수 없습니다" 에러가 과하게 나는지 관찰(스로틀 한계 메모)
- [ ] 그리기 모드에서 살짝 탭(드래그 거의 없음) → 깨진 1점 경로가 생기지 않음

## 9. 권한 거부 경로 (선택 — 별도 실행: 앱 삭제 후 재설치하거나 설정에서 위치 거부)
- [ ] 위치 거부 → 크래시 없이 서울시청 부근으로 폴백

## 10. 의도 일치 최종 판단 (테스트로 못 잡는 부분)
- [ ] 전체 상호작용(모드 토글 → 드래그로 그림 → 첫점=출발/끝점=도착)이 **내가 원했던 UX와 맞는가**

## 참고 — 알려진 한계 (버그 아님, 다음 슬라이스 follow-up으로 합의됨)
- 그린 코스에는 출발/도착 핀이 표시되지 않음(핀은 탭 방식에서만)
- 탭으로 찍은 출발/도착 핀은 초기화/되돌리기로 리셋되지 않음(탭·그리기 혼용 시 잔상)
- 편집(추가/되돌리기)마다 전체 경로를 다시 라우팅 → 긴 세션에서 호출 누적(스로틀 스케일 한계)
- 시뮬레이터 자동화로는 실제 손가락 드래그를 못 돌려, 위 항목들은 실기기에서만 최종 확인됨
