# MVP4 설계 — 그리기 경험 핵심 개선

날짜: 2026-06-26

## 목표

그리기 모드의 핵심 경험 품질을 높인다.
- 그리기 중에도 지도를 자유롭게 이동할 수 있게 한다.
- 내가 그린 형태대로 경로가 라우팅되게 한다.

## 마일스톤 구성

| 순서 | 슬러그 | 핵심 변경 | 사용자가 얻는 것 |
|------|--------|-----------|-----------------|
| 1 | `mkmap-migration` | SwiftUI Map → MKMapView(UIViewRepresentable) | 그리기 중 2손가락 지도 이동 가능, 모드 전환/초기화 불필요 |
| 2 | `drawing-precision` | DrawnPathSampler 누적거리 기반 교체 + 스로틀 에러 감지 수정 | 그린 형태대로 경로 표시, 스로틀 에러 메시지 정상 표시 |

## 마일스톤 1: `mkmap-migration`

### 문제

SwiftUI Map 위 오버레이 제스처는 Map 내부 UIKit 제스처로 전달되지 않는다(hit-test 소유권 문제). 그리기 모드 중 지도가 고정되어 넓은 구간을 한 번에 그릴 수 없다. 모드 전환으로 우회하면 그린 경로가 초기화된다.

### 설계

`MapViewRepresentable`: MKMapView를 UIViewRepresentable로 래핑하는 신규 파일.

**제스처 (핵심 주의사항)**
MKMapView의 기본 패닝은 1손가락이다. "기본 유지"로는 드로우 1손가락과 충돌하므로, 제스처 분리를 명시적으로 구성해야 한다.
- 커스텀 1손가락 `UIPanGestureRecognizer` → 드로우 획 수집 (그리기 모드일 때만 활성화)
- 커스텀 2손가락 `UIPanGestureRecognizer` → 지도 이동 (MKMapView 기본 1손가락 패닝 비활성화 후 대체)
- `drawRecognizer.require(toFail: twoFingerPan)` 으로 우선순위 분리: 1손가락 입력이 2손가락 인식기의 실패 판정 이후에만 드로우로 처리됨
- 줌(핀치): MKMapView 기본 핀치 제스처는 그대로 유지

**오버레이**
- `MapPolyline` → `MKPolyline` + `MKOverlayRenderer`
- Coordinator가 `MKMapViewDelegate`로 처리

**어노테이션**
- `Marker` → `MKPointAnnotation` + `MKAnnotationView`
- 출발(녹색)/도착(빨강) 핀 동일하게 표시
- `UserAnnotation` (파란 현위치 점) → `mapView.showsUserLocation = true` (MKUserLocation 자동 처리)

**카메라**
- `MKCoordinateRegion`을 Binding으로 주고받아 기존 `CameraStateStore` 저장/복원 로직 유지

**변경 범위**
- 신규: `Trace/Pages/CoursePlannerPage/MapViewRepresentable.swift`
- 수정: `Trace/Pages/CoursePlannerPage/CoursePlannerPage.swift` — Map 블록을 MapViewRepresentable로 교체
- 무변경: ViewModel, Domain, Infrastructure 레이어 (ViewModel은 MapKit import 없음 유지)

## 마일스톤 2: `drawing-precision`

### 문제 1: 루프 경로 라우팅 오류

`DrawnPathSampler`가 "마지막 샘플 포인트로부터 직선거리 120m 이상인 점만 보존"하는 방식이라, 원형 획에서 중간 포인트가 거의 다 날아간다. 결과적으로 [시작점, 끝점] 2개만 남아 0.01 km 경로가 나온다.

**수정**
누적거리 기반 샘플링으로 교체: 원본 획을 따라 이동한 누적거리가 120m를 넘을 때마다 포인트를 보존. 원(둘레 300m)이면 시작 + 중간 2개 + 끝 = 4개 포인트가 획 위에 고르게 찍히고, 그 순서대로 라우팅된다.

**한계**: 누적 간격이 120m이므로 둘레 120m 미만의 루프(예: 작은 광장 한 바퀴)는 여전히 [시작, 끝] 2개만 남을 수 있다. 이 경우는 이번 MVP에서 허용하는 한계로 기록.

**샘플러-스로틀 연동 주의**: 누적거리 기반으로 교체하면 루프/곡선 획에서 보존 포인트가 늘어나 MKDirections 구간 호출 수도 증가한다(기존 직선 획 대비 1구간 → 3~4구간). `minSpacingMeters` 값은 정밀도와 스로틀 압박 사이의 균형점이므로, 실기기 테스트에서 실제 스로틀 빈도를 확인하고 필요 시 값을 조정한다(기본 120m는 잠정 값).

변경 범위: `Trace/Domain/CoursePlanning/DrawnPathSampler.swift`

### 문제 2: 스로틀 에러 감지 미작동

`GEOErrorDomain Code=-3`으로 감지 중인데 실기기에서 실제 에러 domain/code가 다를 수 있어 메시지가 표시되지 않는다.

**수정**
실기기에서 빠른 연속 그리기로 스로틀을 유도해 실제 에러 로그 확인 → 정확한 domain/code로 감지 로직 업데이트.

변경 범위: `Trace/Infrastructure/CoursePlanning/MapKit/MapKitCoursePlanningService.swift`

## 검증 계획

### 마일스톤 1: `mkmap-migration` (시뮬레이터 수동 체크리스트)

| 항목 | 통과 기준 |
|------|-----------|
| 1손가락 드로우 | 그리기 모드에서 1손가락으로 획을 그으면 주황 프리뷰 표시 |
| 2손가락 지도 이동 | 그리기 모드에서 2손가락으로 지도가 이동됨 |
| 탭 모드 지도 이동 | 탭 모드에서 1손가락으로 지도가 이동됨 |
| 폴리라인 렌더링 | 경로 계산 후 파란 폴리라인 표시 |
| 마커 표시 | 출발(녹색)/도착(빨강) 핀 정상 표시 |
| 현위치 점 | 파란 현위치 점(UserLocation) 정상 표시 |
| 카메라 복원 | 백그라운드 후 재진입 시 저장된 위치로 복원 |

### 마일스톤 2: `drawing-precision` (유닛 테스트)

`DrawnPathSampler`는 순수 함수이므로 유닛 테스트로 검증한다.

- **테스트 1 (루프 버그 재현)**: 원형 획 좌표(둘레 ≈ 300m)를 직선거리 기반 현행 sampler에 입력 → 결과가 2개임을 확인 (버그 재현)
- **테스트 2 (수정 확인)**: 동일 입력을 누적거리 기반 수정 sampler에 입력 → 결과가 ≥ 3개이며 원래 획 순서를 따름을 확인
- **테스트 3 (직선 획)**: 직선 획에서 기존 동작(간격 유지)이 유지됨을 확인
