# 러닝 시작·종료 손질 실기기 체크리스트 (2026-07-18)

> 이제 러닝 탭에서 "시작"을 누르면 곧바로 뛰는 대신, "삼, 이, 일" 목소리와 함께 3초
> 카운트다운이 나온 뒤에 러닝이 시작됩니다. 종료 버튼을 길게 누를 때는 누르는 동안
> 버튼 테두리에 진행 링이 차오르는 게 보입니다. 목표 거리·시간을 정할 때는 예전처럼
> 숫자를 굴려서 고르는 대신 직접 숫자를 입력하는 방식으로 바뀌었고, 마지막으로 사용한
> 값이 다음에 자동으로 채워집니다. 목소리 안내 문구도 전반적으로 손봤습니다.

빌드/테스트는 전부 통과했습니다(자동 테스트 295개 전체 통과, 회귀 없음). 시뮬레이터에서는
화면 전환과 카운트다운·진행 링·입력 필드 렌더링을 눈으로 확인했지만, 실제 소리·진동·
화면 잠금 동작과 말하는 속도의 체감은 실제 기기에서만 확인할 수 있습니다.

**사용법:** 아래는 총 3개의 "세션"으로 묶여 있습니다. 한 세션 안의 번호는 앞에서부터
순서대로 이어서 수행하면 되고(중간에 앱을 새로 켜거나 하지 않아도 됨), 번호마다 각각
통과/실패를 체크합니다. 다르게 나온 항목은 메모에 무슨 일이 일어났는지 짧게 적어 주세요.

**실제로 걷거나 뛰기 번거로우면:** `docs/qa/trace-qa-5km-straight.gpx` 파일을 준비해뒀습니다.
기기를 케이블로 연결한 채 Xcode → Debug → Simulate Location에서 이 파일을 선택하면,
실기기에서도 제자리에 있는 채로 폰이 정해진 경로를 이동하는 것처럼 인식시킬 수 있습니다
(화면 잠금 중에도 계속 흘러갑니다). 1km당 2분 페이스로 총 10분(5km) 분량이라, 세션
2-2번(1km 지점)까지 2분이면 도달하고 그 뒤로도 8분 여유가 있어 세션 2의 나머지 단계를
느긋하게 진행할 수 있습니다. **파일이 끝(10분)에 도달하면 Xcode가 처음으로 되감아 반복
재생하면서 위치가 순간이동하니, 확인할 것을 다 확인했으면 파일이 끝나기 전에 종료 버튼을
눌러 주세요.** 세션 1·3은 이동이 필요 없어 이 파일과 무관하게 제자리에서 확인 가능하고,
이월 항목(뒤쪽 시나리오 A·B)은 반대로 실제 환경(건물 밀집 지역, 실제 20분 이상 이동)이
확인 목적 자체라 GPX로 대신할 수 없습니다. (방법 상세:
`docs/solutions/workflow-issues/gpx-simulated-location-real-device-qa.md`)

## 핵심 기능 (손으로 수행)

### 세션 1: 시작 카운트다운 4가지 상황

**준비:** 러닝 탭에서 위치 권한을 이미 허용한 상태. 화면을 보면서(잠금 없이) 대기 화면에
있는 상태.

**수행 및 확인:**

**1-1. 정상 완료 — 삼·이·일 발화와 진동**
1. "시작" 버튼을 누른다.
2. 화면과 소리를 함께 지켜본다.

확인할 것: 화면 전체가 어두워지면서 큰 숫자(3, 2, 1)가 하나씩 나타나고, 숫자가 바뀔
때마다 "삼", "이", "일" 목소리와 함께 짧은 진동이 느껴져야 한다. "일" 다음에 카운트다운
화면이 사라지고 실제 러닝이 시작돼야 한다. 확인했으면 종료 버튼을 길게 눌러 러닝을
끝낸다(다음 확인으로 넘어가기 위해).

**결과:** ☐ 통과 ☐ 실패
**메모:**

**1-2. 화면을 끄고 잠가도 발화가 계속되는지**
1. "시작" 버튼을 누르자마자 곧바로 전원 버튼을 눌러 화면을 끈다(잠금 화면으로 전환).
2. 화면이 꺼진 채로 귀를 기울인다.

확인할 것: 화면이 꺼져 있어도 "삼", "이", "일" 목소리가 끊기지 않고 그대로 들려야 하고,
카운트다운이 끝나면 러닝이 정상적으로 시작돼 있어야 한다(화면을 다시 켜서 확인). 확인했으면
종료 버튼을 길게 눌러 러닝을 끝낸다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

**1-3. 화면을 탭하면 취소되는지**
1. "시작" 버튼을 눌러 카운트다운을 띄운다.
2. 숫자가 "2" 정도일 때 어두워진 화면 아무 곳이나 손가락으로 탭한다.

확인할 것: 탭한 즉시 목소리가 뚝 멈추고 카운트다운 화면이 사라지며 원래의 대기 화면으로
돌아가야 한다(러닝이 시작되면 안 됨).

**결과:** ☐ 통과 ☐ 실패
**메모:**

**1-4. 앱을 완전히 새로 켰을 때도 정상 동작하는지 (콜드 스타트)**
1. 최근 사용한 앱 목록에서 이 앱을 위로 쓸어올려 완전히 종료한다.
2. 홈 화면에서 앱 아이콘을 눌러 새로 실행한다.
3. 러닝 탭으로 이동해 "시작" 버튼을 누른다.

확인할 것: 앱을 처음 켠 직후에도 카운트다운 화면과 "삼·이·일" 목소리가 앞서와 똑같이
정상적으로 나와야 한다(첫 실행이라고 소리가 안 나오거나 화면이 이상하면 실패). 확인했으면
카운트다운이 끝난 뒤 종료 버튼을 길게 눌러 러닝을 끝낸다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

### 세션 2: 한 번 제대로 뛰어보기 (속도 체감 + 홀드 링 + 저장까지)

**준비:** 대기 화면에 있는 상태.

**수행 및 확인:**

**2-1. 목표 없이 시작 — 시작 발화**
1. 목표 선택을 "자유"로 둔 채 "시작" 버튼을 눌러 카운트다운이 끝날 때까지 기다린다.

확인할 것: 러닝이 시작되는 순간 짧은 목소리 안내가 나온다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

**2-2. 1킬로미터 지점까지 이동 — 안내와 속도 체감**
1. 1킬로미터 지점까지 걷거나 뛴다(또는 GPX 시뮬레이션으로 대신 진행 — 위 안내 참고).

확인할 것: 1킬로미터를 지나는 순간 목소리 안내가 나온다. 말하는 속도가 너무 빨라서
알아듣기 힘들거나, 반대로 너무 느려서 답답하게 느껴지지 않는지 판단한다. "적당하다" /
"더 빠르게" / "더 느리게" 중 느낀 대로 메모에 적어 달라.

**결과:** ☐ 통과 ☐ 실패
**메모:**

**2-3. 일시정지 → 재개**
1. 화면 안 일시정지 버튼을 누른다.
2. 재개 버튼을 누른다.

확인할 것: 일시정지·재개 각각에서 화면과 동작이 자연스럽게 전환된다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

**2-4. 종료 버튼 살짝 눌렀다 떼기 — 링이 되감기는지**
1. 화면 아래 "길게 눌러 종료" 버튼을 손가락으로 누르고, 1초가 채 되기 전에 손가락을
   뗀다.

확인할 것: 누르는 동안 버튼 테두리를 따라 하얀 선이 채워지다가, 손을 떼면 다시 원래대로
돌아가며 러닝이 끝나지 않아야 한다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

**2-5. 종료 버튼 제대로 길게 눌러 종료**
1. 다시 버튼을 누르고 이번엔 1초 이상 유지한다.

확인할 것: 테두리 선이 끝까지 채워지고, 짧은 진동(완료 느낌)과 함께 러닝이 종료된다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

**2-6. 방금 뛴 기록이 잘 저장됐는지**
1. 기록 목록에서 방금 저장된 기록을 열어본다.

확인할 것: 기록 상세 화면에 거리·시간·경로가 정상적으로 저장돼 있다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

### 세션 3: 목표 거리·시간 직접 입력

**준비:** 대기 화면에서 목표 설정 영역을 "거리" 모드로 바꿔 둔다.

**수행 및 확인:**

**3-1. 값 입력과 키보드 닫기**
1. 숫자 입력 칸에 "5.5" 같은 값을 입력하고 키보드 위 "완료" 버튼을 눌러 키보드를
   내린다.

확인할 것: 숫자 입력 중 키보드가 시작 버튼을 완전히 가리지 않는다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

**3-2. 직전 값 기억(프리필)**
1. 이 값으로 러닝을 시작했다가(카운트다운 중 화면 탭으로 취소해도 됨) 앱을 완전히
   종료했다 다시 켠다.
2. 대기 화면에서 목표를 다시 "거리" 모드로 바꿔본다.

확인할 것: 입력 칸에 직전에 썼던 "5.5"가 이미 채워져 있다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

**3-3. 잘못된 값 입력 — 에러 문구와 시작 버튼 비활성**
1. 입력 칸을 지우고 "0"이나 "abc" 같은 값을 넣어본다.
2. 시간 모드로 바꿔서 "30.5"처럼 소수점이 있는 값을 넣어본다.

확인할 것: 잘못된 값을 넣으면 입력 칸 아래 빨간 안내 문구가 뜨고 "시작" 버튼이 흐리게
바뀌며 눌리지 않는다(값을 다시 올바르게 고치면 버튼이 원래대로 돌아온다).

**결과:** ☐ 통과 ☐ 실패

## 알려진 한계 (버그 아님)
- [x] 목소리 안내 속도(rate) — QA 완료, 반영됨. 카운트다운 첫 숫자("삼")가 뭉개져 들리는
      문제는 오디오 예열 순서 수정으로 해결. km 안내·목표 달성처럼 정보가 많은 문구는

- [x] **재오픈(2026-07-19):** 앱을 완전히 새로 켠 직후 첫 러닝에서만 카운트다운이 다시
      빨라진(뭉쳐 들리는) 느낌 — 두 번째 실행부터는 정상이라는 실사용 QA 피드백. 위 오디오
      예열(세션 활성화)과는 별개로, `AVSpeechSynthesizer`/음성 로딩 자체가 프로세스 최초
      발화에서만 지연이 크다는 플랫폼 특성으로 진단(카운트다운 루프는 시각 타이머 그대로라
      "삼"만 늦게 나오고 "이·일"이 뒤따라 뭉치는 현상). `SpeechVoiceAnnouncer.init()`에서
      앱 시작 시 1회 무음에 가까운(volume 0.01) 발화로 엔진을 미리 예열하도록 수정
      (`.mixWithOthers`로 진행해 배경 음악 덕킹 없음). 실기기 재확인 완료(2026-07-19) —
      콜드 스타트에서도 정상 속도로 확인됨.
