# MVP2 완료 회고 — UX 개선 + 스로틀 완화 (2026-06-23)

## Keep (잘된 것)

- **서브에이전트 분업**: Task 2~5를 각각 서브에이전트로 병렬 파견하여 구현 속도 확보. 테스트 통과까지 확인 후 메인에서 커밋하는 패턴이 안정적이었음.
- **TDD 흐름**: 각 Task마다 테스트 작성 → 실패 확인 → 구현 → 통과 순서를 지켜 regression 없이 진행. 최종 31개 테스트 전부 통과.
- **실기기 피드백 → 백로그 루프**: 실기기 체크리스트(9플로우 36항목) 작성, 사용자 실기기 검증, 피드백을 즉시 백로그에 등록하는 사이클이 자연스럽게 동작.

## Problem (문제)

- **iOS 18.5 @Observable malloc 크래시**: 테스트 실행 시 `malloc: pointer being freed was not allocated` 크래시가 반복되어 이전 세션이 무한 루프에 빠짐. Swift 런타임 버그(swiftlang/swift#87316)로 확인. iOS 26.5 전환 + 블로킹 stub 분리 + `-parallel-testing-enabled NO`로 해결. 이 문제만으로 세션 1개를 소모.
- **xcodebuild 백그라운드 실행 + 파이프**: CLI에서 xcodebuild를 백그라운드로 실행하면서 파이프(`| tail`, `| grep`)를 걸면 출력이 잘림. 이로 인해 프로세스가 좀비로 누적되어 시뮬레이터가 계속 새로 뜨는 현상 발생. 포그라운드 실행 또는 파이프 없는 리다이렉트로 해결.
- **XcodeBuildMCP 지도 탭 불가**: SwiftUI Map의 `SpatialTapGesture`가 UI 스냅샷에 탭 타겟으로 노출되지 않아 시뮬레이터에서 지도 탭 테스트 불가. 탭/그리기 관련 검증은 실기기에 의존해야 했음.

## Surprise (예상 밖)

- **블로킹 stub이 다른 테스트를 죽임**: 레이스 컨디션 테스트용 `CheckedContinuation` 블로킹 stub이 모든 테스트에서 공유되면서, `snappedRoute` 기본 구현이 `route()`를 호출할 때 영원히 블로킹. 테스트 더블 설계에서 "기본 동작은 즉시 반환, 블로킹은 전용 클래스"가 철칙임을 확인.
- **지도 앱 초기 로딩 UX**: 프로덕션 지도 앱들이 마지막 카메라 영역을 디스크에 저장하고 동기 복원하는 패턴을 사용한다는 것. UserDefaults + CLLocationManager.location 캐시 조합이 표준.
