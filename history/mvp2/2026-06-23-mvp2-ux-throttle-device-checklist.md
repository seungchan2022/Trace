# MVP2 UX 개선 + 스로틀 완화 실기기 체크리스트 (2026-06-23)

## 빌드/설치

- [ ] 기기 연결, 자동 서명 팀 확인, Xcode Run 성공
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰 (설정 > 일반 > VPN 및 기기 관리)

---

## 플로우 1: 앱 진입 + 위치

1. [ ] 앱을 처음 실행한다
2. [ ] 위치 권한 팝업이 나타난다 ("'Trace' 앱이 사용자의 위치를 사용하도록 허용하겠습니까?")
3. [ ] **"앱을 사용하는 동안 허용"** 탭 → 현재 위치로 100m 수준 줌인 + 파란 점(내 위치) 표시
4. [ ] 하단 안내: "지도에서 출발지를 선택하세요"
5. [ ] 지도를 다른 곳으로 이동한 뒤, 우하단 **위치 버튼(화살표)** 탭 → 현재 위치로 재중심

## 플로우 2: 탭 모드 — 출발/도착 + 경로

6. [ ] 지도에서 **아무 지점 탭** → 초록 핀(출발) 표시
7. [ ] 다른 지점 탭 → 빨간 핀(도착) 표시 + 경로(파란 선) + 하단에 거리(km)
8. [ ] 또 다른 지점 탭 → 기존 경로 사라지고 새 출발 핀으로 교체

## 플로우 3: 그리기 모드 전환

9. [ ] 상단 **"그리기" 버튼** 탭
10. [ ] 버튼 라벨이 "그리기 중" + pencil.tip 아이콘으로 변경
11. [ ] 상단 배경에 연한 주황색이 보이는지 확인 (매우 연할 수 있음 — 보이는 정도 메모)
12. [ ] 탭 모드의 출발/도착 핀이 사라졌는지 확인
13. [ ] 하단 안내: "경로를 그려주세요"
14. [ ] "되돌리기" 버튼이 나타나는지 확인 (비활성 상태)

## 플로우 4: 그리기로 경로 생성

15. [ ] 지도 위에서 **손가락으로 드래그**하여 경로를 그린다
16. [ ] 드래그를 떼면 잠시 후 경로(파란 선)가 도로에 스냅되어 표시
17. [ ] 경로의 **시작점에 초록 핀(출발)**, **끝점에 빨간 핀(도착)** 표시
18. [ ] 하단에 거리(km) 표시
19. [ ] 추가로 한 번 더 드래그하여 경로 연장 → 거리 업데이트

## 플로우 5: 되돌리기

20. [ ] **"되돌리기" 버튼** 탭 → 마지막 스트로크가 제거, 경로 재계산
21. [ ] 모든 스트로크를 되돌리면 경로·핀·거리 모두 사라짐

## 플로우 6: 모드 전환 시 상태 정리

22. [ ] 그리기로 경로를 만든 상태에서 **"그리기 중" 버튼** 탭 → 탭 모드 복귀
23. [ ] 그리기 결과(경로, 핀, 거리)가 모두 사라지는지 확인
24. [ ] 탭 모드에서 출발/도착 찍은 상태에서 **"그리기" 버튼** 탭 → 그리기 모드 전환
25. [ ] 탭 결과(핀, 경로, 거리)가 모두 사라지는지 확인

## 플로우 7: 초기화

26. [ ] 탭 모드에서 출발 핀을 찍은 상태로 **"초기화" 버튼** 탭
27. [ ] 핀·경로·거리·에러 메시지 모두 사라지는지 확인
28. [ ] 그리기 모드에서 경로를 그린 상태로 **"초기화" 버튼** 탭
29. [ ] 핀·경로·거리 모두 사라지는지 확인

## 플로우 8: 빠른 연속 그리기 (스로틀 테스트)

30. [ ] 그리기 모드에서 **빠르게 3~4회 연속 드래그** (짧은 선을 연달아)
31. [ ] "경로를 계산할 수 없습니다" 에러 없이 최종 경로가 표시되는지 확인
32. [ ] 에러가 나타나면 횟수와 상황 메모

## 플로우 9: 권한 거부

33. [ ] 설정에서 Trace 앱의 위치 권한을 **"허용 안 함"**으로 변경 후 앱 재실행
34. [ ] "위치 권한이 필요합니다" 알럿이 나타나는지 확인
35. [ ] "설정으로 이동" 탭 → 설정 앱으로 이동하는지 확인
36. [ ] "닫기" 탭 → 알럿 닫히고 서울시청 부근(폴백 위치)으로 지도 표시
