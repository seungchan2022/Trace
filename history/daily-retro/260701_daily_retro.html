상황 · 선택지 · 결정

결정
렌더링에서 임의로 구간을 나누는 대신(A안), CourseEditSession.segments 자체가 "1 동작 = 1 세그먼트"이도록 데이터 모델을 통일했다(B안).

왜
세그먼트가 도메인 개념으로 한 번 정해지면 색상·라벨·패널·undo가 전부 같은 배열을 공유해서 서로 어긋날 일이 없다.

대화 기억과 git 워킹 디렉터리는 서로 다른 층이라는 걸 실제로 확인한 사건.

인사이트

💡 생각을 먼저 구조화해야 전달이 쉽다. 오늘 가장 어려웠던 건 코드가 아니라 "내가 원하는 걸 어떻게 표현할까"였다. 결론만 말하지 않고 "그렇게 판단한 근거"를 같이 붙이면, 듣는 쪽이 재확인 없이 바로 움직일 수 있다.

배운 것

  • index 기반 구별 → identity 기반 구별로의 전환 패턴
  • CheckedContinuation 재진입 가드가 "중복 방지"와 "겹친 요청 공유"를 혼동하면 정상 동시 호출까지 막는다
  • MapKit collisionMode가 겹친 어노테이션을 자동으로 숨긴다
  • git 워킹 디렉터리는 대화 세션과 독립적으로 보존된다

느낀 점

세션이 끊겼을 땐 당황했지만, 코드가 그대로 남아있는 걸 확인하고 나니 "대화가 기억하는 것"과 "실제로 존재하는 것"을 구분해서 생각하는 게 훨씬 침착한 방법이라는 걸 깨달았다. 하루 안에 기능 4개 + 버그 3개 + 아카이빙까지 끝낸 건 뿌듯하지만, 표현이 정확하지 않아 재확인이 필요했던 순간들이 아쉬움으로 남는다.
