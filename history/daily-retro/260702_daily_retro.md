## 핵심 의사결정과 이유

- **구간 패널을 화면 높이 40% 상한 + 스크롤 + 채팅 앱 방식 자동 추적으로 설계**: 구간이 많아지면
  패널이 지도를 다 가리는 문제를 상한선으로 막고, 사용자가 스크롤을 올려서 과거 구간을 보고 있을 땐
  자동 스크롤이 방해하지 않도록(채팅 앱의 "새 메시지 자동 스크롤" 관례를 참고).
- **커스텀 핀치 제스처 삭제, 네이티브 줌 복원**: 그리기 모드에서 손가락 1개만 드로우가 쓰므로
  `isZoomEnabled=true`를 그냥 켜도 충돌이 없다는 걸 확인 — 커스텀 구현을 유지할 이유가 없어 삭제.
- **XcodeBuildMCP 테스트 툴을 테스트 실행에 안 쓰기로 규칙화**: 병렬 테스트가 시뮬레이터를 자동
  복제해서 이 프로젝트의 알려진 크래시성 플레이키 테스트가 복제본에서 크래시하면 xcodebuild가
  응답 없이 무한 대기하는 사고를 겪은 뒤, raw `xcodebuild -parallel-testing-enabled NO` 사용을 못 박음.
