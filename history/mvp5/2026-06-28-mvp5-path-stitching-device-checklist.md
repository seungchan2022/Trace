# MVP5 path-stitching 실기기 체크리스트 (2026-06-28)

기능 범위: CourseSegment 세그먼트 모델, 탭·그리기 혼합 경로, 그리기 모드 전환,
2손가락 지도 이동, 핀치 줌, 스트로크 취소/초기화.

---

## 빌드/설치

- [ ] 실기기 연결 → Xcode에서 팀 서명 자동 감지 확인
- [ ] Run(▶) 성공 — 첫 실행 시 **설정 > 일반 > VPN 및 기기 관리**에서 개발자 인증서 신뢰
- [ ] 앱 아이콘이 홈 화면에 정상 표시

---

## 권한

- [ ] 첫 실행 시 위치 권한 팝업 문구가 자연스럽고 적절함
- [ ] "허용" 시: 지도가 현재 위치로 센터링됨
- [ ] "거부" 시: 크래시 없이 서울 기본 위치로 폴백됨, 경고 알림 표시

---

## 탭 모드 — 기본 경로 계획

- [ ] 지도 첫 탭 → 출발지 핀 표시
- [ ] 지도 두 번째 탭 → 도착지 핀 + 도보 경로 폴리라인 + 거리 표시
- [ ] 지도를 다시 탭하면 출발지가 초기화되고 새 출발지로 이동 (경로 폴리라인 유지)
- [ ] 로딩 중 스피너가 자연스럽게 표시/사라짐
- [ ] 한국(실제 테스트 위치) MKDirections 도보 경로가 정상 반환됨

---

## 그리기 모드 전환

- [ ] "경로 찍기" 버튼 탭 → 버튼이 "그리기"로 바뀌고 컨트롤 바 주황색 배경 (현재 상태 표시)
- [ ] 그리기 모드 진입 시 기존 탭 경로가 history에 봉인돼 폴리라인으로 유지됨
- [ ] "그리기" 버튼 재탭 → "경로 찍기"로 복귀, 그린 선이 기존 경로에 이어 붙어 표시됨
- [ ] 모드 전환을 반복(탭→그리기→탭→그리기)해도 누적 경로가 올바르게 유지됨

---

## 그리기 — 1손가락 스트로크

- [ ] 1손가락 드래그 → 실시간 미리보기 선(흰 점선 등) 표시
- [ ] 손가락 뗌 → 도보 경로가 계산되어 기존 경로에 이어 붙은 폴리라인으로 확정
- [ ] 짧은 터치(거의 이동 없음, 2포인트 미만) → 스트로크로 처리되지 않음, 오류 없음
- [ ] 여러 스트로크를 연속으로 그려도 모두 누적됨

---

## 그리기 중 2손가락 지도 이동 (핵심)

- [ ] 그리기 모드에서 2손가락으로 지도 드래그 → 지도가 이동함 (1손가락 스트로크는 취소됨)
- [ ] 2손가락 이동 방향이 화면 상하좌우와 직관적으로 일치함 (부호 반전 검증)
- [ ] 지도 이동 후 1손가락으로 다시 그리기 정상 동작

---

## 그리기 중 핀치 줌

- [ ] 그리기 모드에서 핀치 아웃 → 지도 확대
- [ ] 그리기 모드에서 핀치 인 → 지도 축소
- [ ] 확대/축소 후 그리기 계속 가능

---

## 2손가락 도착 시 스트로크 자동 취소 (멀티터치 가드)

- [ ] 1손가락으로 그리는 중 두 번째 손가락 추가 → 진행 중 스트로크 즉시 취소, 잔선 없음
- [ ] 두 번째 손가락 추가 후 2손가락 패닝으로 자연스럽게 전환됨

---

## 그리기 모드 — Pitch/Rotate 잠금

- [ ] 그리기 모드에서 지도를 기울이거나 회전시키려 해도 변화 없음 (고정됨)
- [ ] 탭 모드에서는 기울기/회전이 다시 작동함

---

## 되돌리기 (Undo)

- [ ] 그리기 모드에서 스트로크 1개 그린 뒤 "되돌리기" → 마지막 스트로크 제거, 이전 상태 복원
- [ ] 모든 스트로크를 되돌리면 history 기반 경로(탭 세션 결과)만 남음
- [ ] 스트로크가 없을 때 "되돌리기" 버튼 비활성(탭 불가)

---

## 초기화 (Clear)

- [ ] "초기화" 버튼 탭 → 탭 세션·그리기 세션 전부 삭제, 지도 핀/폴리라인 모두 제거
- [ ] 아무것도 없을 때 "초기화" 버튼 비활성

---

## 혼합 시나리오 (실사용 플로우)

- [ ] 탭으로 A→B 경로 생성 → 그리기 모드에서 B 근처부터 선 추가 → 탭 모드 복귀 → 경로가 A→B+그린 선으로 이어짐
- [ ] 위 과정에서 총 거리가 화면에 올바르게 표시됨 (터무니없는 수치 없음)
- [ ] 탭→그리기→undo→탭→그리기 혼합 반복 시 크래시 없음

---

## 제스처/입력 감

- [ ] 1손가락 드래그 반응이 자연스럽고 지연 없음
- [ ] 2손가락 패닝이 끊기지 않음
- [ ] 핀치 줌 부드러움
- [ ] 버튼 탭 반응 즉시

---

## 견고성 / 엣지 케이스

- [ ] 네트워크 없이 스트로크 그리기 시도 → 에러 메시지 표시, 크래시 없음
- [ ] 스트로크를 빠르게 연속으로 여러 개 그려도 중간 크래시 없음
- [ ] 앱을 백그라운드로 보냈다가 복귀해도 그리기 상태 유지됨 (또는 자연스럽게 리셋)
- [ ] 메모리 경고 시뮬레이션 시 크래시 없음 (선택)

---

## 의도 일치

- [ ] 탭과 그리기를 섞은 경로 계획이 내가 원했던 UX와 부합하는가
- [ ] 2손가락 지도 이동이 "손가락 방향 = 지도 이동 방향"으로 자연스러운가
- [ ] 불일치·개선점이 있으면 `docs/backlog.md`에 기록

---

## 알려진 한계 (버그 아님)

- `recognizer.state = .cancelled` — UIKit 내부 상태 조작, 일부 기기에서 미묘한 차이 가능 (backlog)
- clear 버튼 300ms disabled window — 스트로크 확정 debounce로 인한 의도된 지연 (backlog)
- MKDirections 60초/50요청 스로틀 — 빠른 스트로크 반복 시 "요청이 많아 잠시 후 다시 시도" 메시지 정상
