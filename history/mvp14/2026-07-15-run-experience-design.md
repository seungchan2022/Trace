# MVP14 — 기본 러닝 경험 완성 설계

- 날짜: 2026-07-15
- 상태: 킥오프 브레인스토밍 확정 (사용자 승인)
- 우산: MVP14 (`docs/roadmap.md`) — 마일스톤 3개 / 사이클 3개
- 선행: MVP13 (자유 러닝 트래킹 + 기록, `history/mvp13/`)

## §0. 배경과 목표

MVP13으로 러닝 기둥의 뼈대(시작→GPS 트래킹→종료 요약→자동 저장→기록 목록/상세)가 섰다.
그러나 일반 러닝앱이라면 당연히 있는 것들이 아직 없다: 일시정지가 없어 신호 대기 시간이
기록에 그대로 들어가고, km별 페이스(스플릿)를 볼 수 없고, 뛰는 중 화면을 보지 않으면
아무 정보도 얻을 수 없고, 목표 없이 "무작정 시작"만 가능하다.

MVP14는 이 네 가지를 채워 러닝 기둥을 독립적으로 완성한다. **코스 연동(코스 골라 뛰기 +
계획 vs 실제 비교)은 다음 MVP** — 이번 범위가 아니다.

**설계 상수: "러닝 중 = 화면 꺼짐/백그라운드가 기본 상황"** (사용자 명시, 2026-07-15).
모든 기능이 주머니 속에서 동작해야 하며, 실기기 QA도 이 시나리오를 중심으로 짠다.
MVP13에서 백그라운드 GPS 스트림은 이미 검증됐고, 이번에 새로 검증할 것은
"백그라운드에서 소리가 나는가"다.

## §1. 확정 결정 (킥오프 브레인스토밍, 2026-07-15)

| # | 결정 | 선택 | 기각 대안 |
|---|---|---|---|
| 1 | MVP 방향 | 러닝 경험 완성 묶음 | 오디오+코스연동(너무 큼), 오디오 단독(너무 작음) |
| 2 | 일시정지 | **수동만** | 자동 일시정지 — GPS 노이즈 오판 튜닝·QA 부담. 수동 위에 additive로 나중에 얹을 수 있음. "수동 누르는 걸 까먹는다" 피드백이 나오면 백로그 트리거 |
| 3 | km 경계 발화 내용 | **거리 + 총 시간 + 전체 평균 페이스** | 직전 km 페이스 — 일상 러닝 거리에선 평균이 더 직관적(사용자: 나이키런 방식에 이미 익숙, 평균만으로도 페이스 변화 감지 가능). 직전 km 페이스는 기록 상세 스플릿 표에서 확인. 엔진이 두 값을 다 내므로 나중에 발화 한 줄 추가로 전환 가능 |
| 4 | 발화 시점 | **km 경계 + 상태 전환**(시작/일시정지/재개/종료 요약) | km 경계만 — 화면 없이 "버튼이 눌렸나" 확인 불가, 특히 새로 생기는 일시정지와 궁합 문제 |
| 5 | 음악 공존 | **덕킹**(`.duckOthers`) — 발화 순간만 음악 볼륨을 낮췄다 복원 | 음악 일시정지(매 km 뚝뚝 끊김), 그냥 겹침(안내가 묻힘) |
| 6 | 목표 설정 | **3모드: 자유 / 거리 / 시간.** 달성해도 트래킹 계속(초과분 기록), 종료는 사용자가 | 거리만 — 시간 목표는 타이머 비교라 사실상 공짜, 뺄 이유 없음 |
| 7 | km 안내 적용 범위 | **모드 무관 공통** — 자유 러닝에도 항상. 목표 러닝은 그 위에 목표 안내(절반/달성)가 추가 | — (사용자 명시) |
| 8 | 일시정지 진입점 | **앱 내 트래킹 화면 버튼만** | 잠금화면(Live Activity) 일시정지/재개 버튼 — 스펙 리뷰에서 "화면 꺼짐 상수와 충돌" 지적됐으나, 사용자 판단으로 이연(2026-07-15): 일시정지 사용 빈도가 낮고, 쓰는 상황이면 어차피 폰을 꺼내는 상황이라 잠금 해제 마찰 감수 가능. 인터랙티브 버튼은 additive라 나중에 얹기 쉬움 — §6 트리거 참고 |

## §2. 마일스톤 · 사이클 구성 (3개, 순서 고정)

| 사이클 | 마일스톤 | 무게 | 순서 근거 |
|---|---|---|---|
| 1 | **run-pause-resume** | 표준 | 시간·페이스의 *정의*를 바꾸고(멈춘 시간 제외) 저장 스키마를 건드림 — 스플릿·오디오가 이 정의를 소비하므로 반드시 첫 번째 |
| 2 | **run-splits-audio** | 표준 | 스플릿과 오디오가 km 경계 엔진을 공유하므로 한 사이클. 새 인프라(오디오 세션·TTS·Background Mode) 도입 |

## §3. 아키텍처

기존 패턴(page-first MVVM, port-and-adapter, `RunSession` 소비자 원칙) 유지. 새 타입의
격리는 프로젝트 기본값(기본 nonisolated + UI/상태 타입만 명시 `@MainActor`)을 따른다.

### 3.1 일시정지/재개 (사이클 1)

- `RunSession.State`에 `paused` 추가: `tracking ↔ paused`, `paused → summary`(일시정지 중 종료 허용).
- **GPS 스트림은 일시정지 중에도 유지**하되 샘플을 적산하지 않는다. 배터리 소폭 손해
  대신 **재개가 즉각적**(재획득 대기 없음). 스트림을 껐다 켜면 재개 시 acquiring 상태가
  다시 필요해져 상태기계가 복잡해진다.
- **거리**: 일시정지 전 마지막 샘플 ↔ 재개 후 첫 샘플 사이 거리는 적산하지 않는다
  (멈춘 동안 이동해도 거리에 안 들어감).
- **시간**: 경과 시간 = 벽시계 시간 − 일시정지 구간 합. 페이스·스플릿·목표 판정 전부
  이 "활동 시간" 기준.
- **UI**: 트래킹 화면에 일시정지/재개 버튼 (배치·스타일은 UI 디테일 — 자동 진행). 일시정지
  진입점은 이 버튼뿐(결정 8 — 잠금화면 버튼은 이연).
- **장시간 일시정지는 무대응**: 몇 시간을 방치해도 스트림 유지·세션 유지(리마인더·자동 종료
  없음). "일시정지는 짧다"는 전제의 의도적 단순화 — 배터리 불만 피드백을 트리거로 재검토(§6).
- **Live Activity**: 일시정지 표시(배지 + 시간 정지). 현재 위젯은 고정 `Attributes.startedAt`
  기반 `Text(timerInterval:)`로 시간을 그리므로 불리언 추가만으로는 정지·재개를 표현할 수 없다
  — 타이머 기준 구간(누적 일시정지만큼 보정된 시작 시각, 일시정지 시각)을 `ContentState`로
  옮긴다 (additive).

### 3.2 km 스플릿 엔진 (사이클 2)

- `RunSplitCalculator` — Domain 순수 로직. 입력: 샘플 스트림 + 일시정지 구간. 출력:
  km별 `(구간 시간, 구간 페이스)` 배열 + 마지막 미완성 구간. 구간 시간은 일시정지 제외.
- **라이브와 소급이 같은 코드**: 트래킹 중에는 증분으로 km 경계 감지(경계를 넘는 순간
  이벤트), 기록 상세에서는 저장된 샘플 전체로 일괄 계산.
- **소급 적용**: 기록이 원시 샘플 5필드를 전부 저장하므로(MVP13 §스키마) 이미 저장된
  과거 러닝에도 스플릿 표가 그대로 나온다. 과거 기록엔 일시정지 구간이 없음(빈 배열) —
  자연스럽게 동작.
- 기록 상세 UI: km별 페이스 표(막대 등 시각화는 UI 디테일).

### 3.3 음성 안내 (사이클 2)

- `VoiceAnnouncerProtocol`(Domain) + `AVSpeechSynthesizer` 어댑터(Infrastructure) —
  기존 port-and-adapter 관례. 한국어 시스템 보이스, 기본 속도.
- **오디오 세션은 발화하는 순간에만 활성화**: `.playback` + `.duckOthers`로 활성화 →
  발화 → 비활성화는 개별 utterance의 `didFinish`마다가 아니라 **대기 중인 발화가 더 없을
  때(큐 소진 시점)만** `.notifyOthersOnDeactivation`으로 수행 → 음악 볼륨 복원. 연속 발화
  (km 경계+목표 달성 동시)에서 볼륨이 복원됐다 다시 내려가는 플랩을 막기 위함.
- **Background Modes에 `audio` 추가** — 화면 꺼진 채 발화가 나오기 위한 필수 조건
  (location은 MVP13에서 이미 추가됨).
- **발화 큐 직렬화**: km 경계와 목표 달성이 동시에 걸리면 순서대로 읽는다
  (AVSpeechSynthesizer 자체 큐 활용).
- `RunAudioCoach`(Application) — `RunSession`의 상태 전환·km 경계·목표 진행을 구독해
  문장을 조립하고 `VoiceAnnouncer`에 넘기는 소비자. MVP13의 "기능 추가 = `RunSession`
  소비자 추가" 원칙 그대로 — 세션 자체는 오디오를 모른다.
- 발화 문구(초안 — 구현 중 다듬기 가능):
  - 시작: "러닝 시작"
  - km 경계: "3킬로미터. 총 시간 18분 30초. 평균 페이스 6분 10초"
  - 일시정지 / 재개: "일시정지" / "재개합니다"
  - 종료: "러닝 종료. 총 5.2킬로미터, 31분 40초, 평균 페이스 6분 5초"
- 음성 on/off 토글은 이번 범위 밖(YAGNI) — 불편 피드백이 나오면 백로그 트리거로 추가.

### 3.4 목표 설정 (사이클 3)

- `RunGoal` 값 타입: `.open` / `.distance(미터)` / `.time(초)`.
- 대기 화면에서 모드 선택 + 값 입력(프리셋/피커 등 입력 UI는 디테일 — 자동 진행).
- `RunSession`이 목표를 보유하고 진행률·절반·달성 이벤트를 낸다(각 1회만). 발화는
  `RunAudioCoach`가: 절반 "절반 왔습니다", 달성 "목표 달성! 5킬로미터, 29분 10초".
- **달성 후에도 트래킹 유지** — 안내만 하고 종료는 사용자가. 초과분도 그대로 기록.
- 트래킹 화면에 목표 진행률 표시(형태는 UI 디테일).

## §4. 저장 스키마 (additive만 — 마이그레이션 리스크 최소)

러닝 스토어는 코스와 별도 파일(MVP13 결정)이라 lightweight migration으로 충분.

1. **일시정지 구간**: `SavedRun`에 일시정지 이벤트(시작·끝 시각 쌍 배열, Codable `Data`)
   추가. **샘플 시각 간격에서 파생 불가**(GPS 끊김과 구분 안 됨)이므로 "파생 가능하면
   저장하지 않는다" 원칙의 예외가 아니라 명시 저장 대상이 맞다.
2. **캐시 컬럼 `duration`**: 의미를 "일시정지 제외한 활동 시간"으로 확정. 기존 기록은
   일시정지가 없으므로 값 변화 없음.
3. **목표**: 기록에 목표 타입·값(옵셔널) 저장 — 기록 상세에 "목표 5km" 표시용.
4. **스플릿은 저장하지 않는다** — 샘플에서 언제든 파생 가능(기존 원칙 준수).

## §5. 테스트 · QA

- **사이클 2 첫 태스크 = 백그라운드 오디오 스파이크**: 이번 MVP의 핵심 미지수("백그라운드에서
  소리가 나는가")를 사이클 말 QA까지 미루지 않는다 — Background Modes `audio` 추가 + 화면
  잠금 + 음악 재생 상태에서 TTS 1회 발화로 덕킹·볼륨 복원까지 실기기에서 먼저 확인한 뒤
  본 구현에 들어간다. 여기서 문제가 나오면 세션 수명주기 설계를 산출물 없이 되돌릴 수 있다.
- **단위 테스트**: 스플릿 계산(경계 분할·미완성 잔여 구간·일시정지 제외), 일시정지
  시간/거리 제외 계산, 목표 판정(절반·달성 각 1회만), 발화 문구 조립(거리·시간·페이스
  포맷). `VoiceAnnouncer`는 프로토콜 뒤라 코치 로직은 페이크로 검증.
- **실기기 QA (필수)**: **주머니 시나리오 중심** — 화면 꺼진 채: km 안내가 나오는가,
  음악 위에서 덕킹이 동작하고 볼륨이 복원되는가, 일시정지/재개 발화가 나오는가,
  Live Activity에 일시정지가 반영되는가. 체크리스트는 사이클별
  `docs/qa/YYYY-MM-DD-<slug>-device-checklist.md` (시나리오 카드 형식).
- **밀린 확인 3건 동반 수행** (킥오프 합의, 2026-07-15): 이번 실기기 QA 라운드에서
  ① 강제종료 후 잠금화면 카드 정리(MVP13 시나리오 10) ② 20분+ 러닝 배터리 체감
  ③ 건물 밀집지 GPS(기회가 되면)를 함께 확인한다. (`docs/backlog.md` MVP13 QA 섹션)

### 플랜 단계에서 확정할 것 (스펙 리뷰에서 이연, 2026-07-15)

- `paused` 상태와 `RunSession.isActive`·스트림 가드·종료 경로의 관계 — 사이클 1 플랜에서
  상태기계 전이표로 확정
- 일시정지 중 종료(`paused → summary`) 시 열린 일시정지 구간을 종료 시각으로 닫는 규칙 —
  스플릿 계산기 입력 계약에 명시
- 백그라운드에서 오디오 세션 활성화 실패(통화 중 등) 시 발화를 건너뛸지 재시도할지 —
  사이클 2 플랜에서 결정

## §6. 범위 밖 (명시적 제외)

- 코스 연동(코스 골라 뛰기 + 계획 vs 실제 비교) — 다음 MVP
- 자동 일시정지 — 트리거: "수동 누르는 걸 까먹는다" 피드백
- 잠금화면(Live Activity) 일시정지/재개 인터랙티브 버튼 — 트리거: "잠금 해제하고 일시정지하는
  게 번거롭다"는 실사용 피드백 (결정 8, 2026-07-15)
- 장시간 일시정지 대응(리마인더 발화·자동 종료) — 트리거: 일시정지 방치로 인한 배터리 불만
  피드백
- 음성 on/off 토글, 발화 빈도 설정(0.5km 단위 등) — 트리거: 불편 피드백
- 직전 km 페이스 발화 — 트리거: 평균만으로 부족하다는 실사용 판단
