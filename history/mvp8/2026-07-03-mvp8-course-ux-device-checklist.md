## 결과 요약 (2026-07-03 실기기 QA 완료)

핵심 기능(패널 스크롤/높이 제한, 핀치 줌 복원, 겹침 오프셋)은 모두 실기기에서 정상 동작 확인됨.
겹침 오프셋은 임시 디버그 로그로 `maxDisplacementMeters≈4.0`(설계값 일치) 직접 확인.
발견된 버그 1건(스크롤 인디케이터 패딩)은 이번 세션에서 수정 완료. 그 외 발견 사항은 모두
`docs/backlog.md` "MVP8 (2026-07-03) 실기기 QA 피드백" 절에 상세 분석과 함께 기록됨.

## 마일스톤 1 — 구간 패널 + 핀치 줌

- [x] 구간 10개 이상 만들었을 때 패널이 화면(지도 높이) ~40%에서 멈추고 그 안에서 스크롤되는지 — 확인됨. 단 스크롤 인디케이터가 콘텐츠와 같이 안쪽으로 밀리는 버그 발견 → 이번 세션에서 수정 완료
- [x] 최신 구간 근처를 보고 있을 때 새 구간을 추가하면 목록이 계속 최신을 따라가는지 (도착 지점/append) — 정상
- [x] 옛 구간을 스크롤해서 보는 중에 새 구간이 추가되면 보던 위치가 유지되는지 — 정상(append). **단 출발 지점(prepend) 추가 시엔 위치가 밀리는 현상 발견 → 백로그 등록**
- [ ] 패널을 접었다 다시 펼쳤을 때 이전에 보던 위치로 복원되는지 (최신 근처였다면 최신으로) — 별도로 테스트 안 함
- [ ] **화면이 큰 기기(Pro Max급)에서 40% 상한 안에 짧은 행이 7~8개 이상 보일 때...** — Pro Max 실기기가 없어 테스트 불가, 보류
- [x] 그리기 모드에서 두 손가락 핀치가 손가락 사이 지점을 기준으로 자연스럽게 줌되는지 — 동작은 하나 체감상 뚜렷한 개선은 못 느낌(버그 아님, 기록만)
- [x] 더블탭 줌이 그리기 시작과 간섭하지 않는지 — 별도 구현이 아니라 `isZoomEnabled=true`에 딸려오는 네이티브 동작(핀치 복원의 부수 효과)이라고 설명함, 간섭 보고 없음
- [x] 두 손가락 탭 줌아웃이 커스텀 2손가락 pan과 간섭하지 않는지 — 천천히 하면 정상, **빠르게 연속 시도하면 충돌 있음 → 백로그 등록**

## 마일스톤 2 — 겹치는 경로 오프셋

- [x] 왕복 코스(같은 길을 되돌아오는 코스)에서 두 선이 나란히 보이고, 각 거리 라벨이 자기 선 위에 있는지 — **정상 동작 확인** (디버그 로그로 `maxDisplacementMeters≈4.0` 확인). 처음엔 안 갈라지는 것처럼 보였으나, 원인은 알고리즘이 아니라 탭/그리기의 자동 연결 로직이 짧은 연결 구간을 만들어서였음(상세: 백로그 "탭 모드로는..." 항목)
- [ ] 왕복 코스에서 도착 핀과 선 끝이 일치하는지 — 별도 확인 안 함. 대신 **왕복(prepend) 시 출발/도착 핀이 실제 뛴 순서와 반대로 붙는 문제**를 발견 → 백로그 등록(이번 마일스톤 범위 밖, 브레인스토밍 필요)
- [ ] **왕복 코스의 turnaround 지점에서 가는 길과 오는 길 사이에 작은 시각적 틈이 보이는지** — 별도로 테스트 안 함
- [ ] 코스 전체를 조망하는 축소 배율에서 라벨-선 색 어긋남이 재현되는지 — 별도로 테스트 안 함
- [ ] 줌 배율을 바꿔가며 두 선이 한 줄로 합쳐 보이기 시작하는 시점 기록 — 별도로 테스트 안 함
- [ ] **짧은 겹침 구간에서 목표 오프셋(4m)까지 도달하지 못하고...** — 별도로 테스트 안 함
- [ ] 겹치는 스트로크가 확정되는 순간 즉시 스냅이 거슬리는지 — 별도로 테스트 안 함
- [ ] 좁은 골목·도로변 산책로에서 거짓 양성이 있는지 — 별도로 테스트 안 함
- [ ] 구간을 추가(attach)한 직후 프레임 히치가 느껴지는지 — 별도로 테스트 안 함
