## 결과 요약 (2026-07-04, 3차례 QA 후 최종 상태)

**수정 완료, 실기기 재확인까지 끝남:**
- **경유점이 거리 라벨을 가끔 가림** — 두 번의 시도(zPosition, 추가 순서 재정렬)가 모두 실기기에서 재발 →
  근본 원인은 애노테이션끼리의 z-order가 애초에 안정적으로 보장 안 된다는 것으로 파악, 경유점을 애노테이션이
  아니라 오버레이로 완전히 옮겨서 "오버레이는 항상 애노테이션보다 아래"라는 MapKit 구조적 보장을 이용하도록
  재구현(commit `fb1069f`). 실기기 재확인 완료.
- **거리 라벨 위치가 랜덤해 보임** — 경유점 재확인 중 사용자가 추가로 발견. 라벨 좌표가 배열 인덱스 절반
  (`coords.count/2`)이라 라우팅 좌표 밀도가 불균일할 때 실제 경로 중간과 무관한 위치에 찍혔음. 누적 실거리
  기반 보간(`midpointAlongPath`)으로 교체(commit `117f821`). 실기기 재확인 완료.

**수정 완료, 재확인 불필요(코드 검증 완료):**
- 그리기 모드 반전(prepend), 핀 선택 애니메이션, 병합 배지 위치 — 아래 각 절 참고.

**의도적으로 보류 → `docs/backlog.md`에 다음 MVP 후보로 등록:**
- **그리기 모드 "출발점 근처" 판정 정밀도** — 디버그 로그로 확인: 코드는 설계대로 정확히 동작 중(버그
  아님). 사용자가 육안으로 "근처"라 여긴 지점이 실제로는 51m 떨어져 있어 20m 임계값을 넘은 것. 임계값을
  단순히 올리는 안은 "경계 문제를 옮길 뿐 근본 해결이 아니다"라는 이유로 기각 — 탭 모드처럼 화면 픽셀
  기준으로 재설계하는 게 맞다고 판단, 다음 MVP에서 제대로 기획.
- **탭모드 더블탭-줌 시 점이 함께 찍힘** — 자체 디바운스로 "두 번째 탭"은 막았지만 첫 번째 탭은 즉시
  반영되어 점이 하나는 찍힘. 완전 차단하려면 모든 탭에 ~0.3초 지연이 필요한데 이 트레이드오프는 받아들이지
  않기로 함(다른 지도 앱은 지연 없이 가능한 것을 확인) — 다음 MVP에서 방법을 제대로 조사.

## 핵심 기능 — attach 규칙 (Task 1)

- [x] 탭 모드에서 A→B→C 순서로 이어 그리면 정상적으로 누적되는지 — 문제 없음
- [x] 그리기 모드에서 출발점 근처에서 시작해 바깥쪽으로 그리면(뒤로 연장) 코스 시작이 바뀌는지 — 코드 동작
      자체는 디버그 로그로 정상 확인됨(`b5e130e`). 단, "근처"의 정밀도(20m)가 육안 기대와 안 맞을 수 있음 —
      `docs/backlog.md` "그리기 모드 출발점 근접 판정 정밀도" 참고, 실사용 시 20m 이내로 바짝 붙여서
      그려야 반전이 트리거됨을 감안할 것
- [x] 도착점 근처에서 시작해 출발점 쪽으로 되짚어 그리면(왕복) 반전되지 않고 그대로 append되는지 — 정상
- [ ] 코스와 전혀 동떨어진 곳에서 새 구간을 그리면 도착점에서 자동 라우팅(gap)으로 연결되고 그린 방향
      그대로 붙는지

## 핵심 기능 — 출발핀 탭 왕복 (Task 4, 5)

- [x] 열린 코스 상태에서 지도 하단(statusPanel)에 "출발핀을 탭하면 왕복 완성" 힌트가 보이는지
- [x] 출발핀을 탭하면 도착점에서 출발점까지 자동으로 라우팅되어 왕복이 완성되는지
- [ ] 도착핀을 탭하면 아무 일도 일어나지 않고 "이미 도착점입니다" 안내가 뜨는지
- [ ] 왕복 완성 후(닫힌 코스) 병합 핀을 탭하면 "이미 닫힌 코스입니다" 안내가 뜨는지
- [ ] **24pt 히트 반경이 실제 손가락 터치에서 체감상 적절한지** — 시뮬레이터로 검증 불가, 실기기 전용 항목
- [ ] 지도를 확대/축소한 상태에서도 핀 히트 판정이 일관되게 느껴지는지

## 핵심 기능 — 닫힌 코스 병합 핀 (Task 6)

- [x] 왕복을 완성하면 출발/도착 핀이 하나로 합쳐지고 체크무늬 배지가 붙는지 — 배지 위치 수정 후 확인됨
- [ ] 병합 핀 배지 크기·모양이 지금(20pt) 상태로 괜찮은지 — 필요하면 추후 다시 조정 가능, 지금은 보류
- [ ] **라우팅 스냅 오차로 인해 왕복을 완성했는데도 병합 핀이 안 뜨고 출발/도착 핀이 분리된 채로 남는 경우가
      있는지** — 코드리뷰 Important 발견, 아직 실기기 재현 안 됨. 지속 관찰 필요

## 핵심 기능 — 경유점 마커 (Task 7)

- [x] 구간을 3개 이상 이어 붙였을 때 각 구간 경계마다 작은 흰 점이 보이는지
- [x] 경유점이 거리 라벨을 가리지 않는지 (항상 라벨이 위) — 두 차례 실패(zPosition, 추가 순서) 후
      애노테이션→오버레이 구조 전환(`fb1069f`)으로 재확인 완료, 해결됨
- [x] 거리 라벨이 구간의 실제 시각적 중간 지점에 위치하는지 — 재확인 도중 추가로 발견된 버그(배열 인덱스
      절반이 실거리 절반과 무관했음), 누적 거리 기반 보간으로 수정(`117f821`), 재확인 완료, 해결됨

## 핵심 기능 — redo (Task 2, 3)

- [ ] 되돌리기(undo) 후 "앞으로" 버튼이 활성화되고, 누르면 원래 상태로 복원되는지
- [ ] prepend로 붙은 구간을 undo→redo했을 때 같은 자리(맨 앞)로 정확히 복원되는지
- [ ] undo→redo 후 새 구간을 추가하면 "앞으로" 버튼이 다시 비활성화되는지 (미래 무효화)

## 제스처/입력 감

- [x] 핀 선택 애니메이션(확대+흔들림)이 더 이상 안 나오는지 — `isEnabled = false` 적용 후 확인됨
- [ ] 핀 탭과 일반 지도 탭(코스 연장)이 서로 헷갈리지 않고 의도대로 구분되는지
- [x] 탭모드에서 화면을 빠르게 두 번 탭(더블탭 줌)해도 핀이 두 개까지는 안 찍히는지 — 자체 디바운스로
      "두 번째 탭"은 막음. 단 **첫 번째 탭 1개는 여전히 찍힘** — 완전 제거는 의도적으로 보류
