작성일: 2026-07-01

## Keep (잘 된 것)

**segmentIndex와 colorKey 분리**
undo가 prepend된 최근 구간을 못 지우는 버그를 고치면서, "배열상 위치"(segmentIndex, 선택 하이라이트 매칭용)와
"attach 생성 순서"(colorKey, 색상 identity)를 서로 다른 개념으로 명시적으로 분리했다. 두 개념을 하나의 index로
뭉뚱그렸던 게 원래 버그의 뿌리였고, 분리한 뒤로는 prepend·undo·색상이 서로 독립적으로 동작한다.

**ContinuationBroadcaster로 재진입 버그를 테스트 가능하게 분리**
`CoreLocationService`는 `CLLocationManager`를 직접 감싸는 Infrastructure 어댑터라 원래 유닛 테스트가 전혀 없었다.
겹친 위치 요청을 재진입 거부 대신 브로드캐스트하도록 고치면서, 그 로직만 `ContinuationBroadcaster<Value>`라는
프레임워크 독립적인 작은 타입으로 뽑아냈다. 덕분에 실기기에서는 타이밍 의존적이라 재현이 불안정한 버그를
결정적인 유닛 테스트 4개로 고정할 수 있었다.

**커밋을 실제 의미 단위로 정확히 분리**
undo/색상 안정성, (C) 핀 collision, (D) 위치 버튼 3개 버그가 파일 몇 개에 걸쳐 뒤섞여 있었는데, `git add -p`로
훅(hunk) 단위까지 골라내 세 개의 독립 커밋으로 나눴다. 각 커밋이 빌드·테스트를 온전히 통과하는 상태를 유지했다.

## Problem (아쉬운 것)

**MVP가 킥오프 시점에 roadmap에 등록되지 않음**
`docs/agent-rules/workflow.md`는 "MVP를 먼저 세우고 마일스톤을 채운다"(top-down)를 원칙으로 하는데, 이 MVP는
`docs/roadmap.md`에 전혀 등록되지 않은 채로 진행됐다. 완료 시점에야 소급으로 MVP7을 등록하고 아카이빙해야 했다.

**plan.md 체크박스가 실행 중 갱신되지 않음**
Task 1~5가 실제로는 각각 커밋까지 끝났는데도 plan 파일의 체크박스는 전부 미체크 상태로 남아 있었다. 실행
당시 실시간으로 체크하지 않아서, 완료 여부를 커밋 로그를 역추적해 재구성해야 했다(소급 확인 노트로 처리).

**세션 단절로 대화 맥락 소실**
작업 도중 `/clear`·`/exit`로 세션이 끊겨 이전 대화의 의사결정 맥락이 사라졌다. git diff·커밋 로그·backlog·QA
체크리스트 문서를 교차 확인해서만 "무엇을 하다 멈췄는지"를 재구성할 수 있었다 — 코드/문서 상태는 보존되지만
대화 맥락은 별개로 소실될 수 있다는 걸 실감했다.

## Surprise (예상 밖)

**세션이 끊겨도 uncommitted 작업은 그대로 남는다**
`/clear`·`/exit`가 대화 기억만 지울 뿐 git 워킹 디렉터리에는 전혀 영향을 주지 않는다는 사실을, 이번에 실제로
확인했다. "세션이 사라졌다"는 불안과 "코드가 사라졌다"는 완전히 다른 문제였다.

**MapKit 어노테이션 충돌(collision)이 핀을 통째로 숨긴다**
거리 라벨 annotation과 출발/도착 핀이 짧은 거리에서 겹치면, MapKit이 기본 `collisionMode`로 핀을 자동으로
숨겨버린다. "마커가 안 보인다"는 QA 리포트의 원인이 라우팅 로직이 아니라 순수 렌더링 레이어의 충돌 처리였다.

**단일 continuation 재진입 가드가 정상적인 동시 호출까지 막았다**
`CoreLocationService.currentLocation()`의 재진입 가드는 "중복 호출 방지"를 의도했지만, 실제로는 정상적인
동시 호출(부트스트랩 + 버튼)까지 실패시켰다. 재진입 방지와 "겹친 요청은 같은 결과를 공유해야 한다"는 서로 다른
요구사항인데 하나의 메커니즘으로 처리하려던 게 근본 원인이었다.

## 마일스톤별 핵심 의사결정

| 마일스톤/버그 | 결정 | 이유 |
|---|---|---|
| A. 탭 자동 연결 | 세그먼트가 하나 이상이면 탭 1번만으로 가까운 끝점에 자동 연결 | 최초 2탭 흐름은 유지, 이어붙이기부터만 UX 단순화 |
| A-2. 그리기 스트로크=세그먼트 | 스트로크마다 즉시 `session.attach()`, 누적 버퍼 제거 | 탭 모드와 동일한 attach 계약으로 통일해 undo/색상 로직 재사용 |
| B. 구간 색상+거리 라벨 | `SegmentPolyline` 서브클래스에 색상 인덱스를 실어 오버레이별로 렌더링 | 세그먼트별 독립 `MKPolyline`이 있어야 색상·라벨을 분리할 수 있음 |
| E. 실시간 구간 패널 | `selectedSegmentIndex`로 패널↔지도 연동 | 패널의 색상 스와치와 지도 폴리라인이 같은 소스(팔레트)를 공유 |
| undo/색상 버그 | `Entry{id, order, segment}` 도입, `segmentColorKeys` 분리 노출 | 공간순(segments)과 시간순(attach order)이 prepend 시 갈라지는 걸 구조적으로 분리 |
| (C) 도착 마커 미표시 | `displayPriority = .required`, `collisionMode = .none` | 핀은 최대 2개뿐이라 성능 영향 없이 항상 표시 강제 가능 |
| (D) 위치 버튼 오작동 | `ContinuationBroadcaster<Value>` 도입 | 재진입 거부 대신 겹친 요청을 하나의 결과로 fan-out |

## 남은 기술부채

- **실시간 구간 패널 무한 확장** — 구간이 늘어날수록 패널 목록 높이가 계속 커짐. 최대 높이+스크롤 필요 (`backlog.md`).
- **겹치는 경로 렌더링 방식** — 경로가 겹치면 최신 구간이 이전 구간을 덮어씀. 오프셋/스타일 구분 등 설계 논의 필요, brainstorm 대상 (`backlog.md`).
- **그리기 모드 핀치 줌 UX** — MVP4부터 이어진 open 항목, 내장 MKMapView 핀치 복원 검토 (`backlog.md`).
