# MVP3 — UX 개선 + 스로틀 강화 실기기 체크리스트 (2026-06-25)

## 빌드/설치
- [ ] 기기 연결, 자동 서명 팀 확인, Xcode Run 성공
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰(설정 > 일반 > VPN 및 기기 관리)

## 권한
- [ ] 위치 권한 팝업 문구 확인, 허용 경로 동작
- [ ] 거부 경로 동작 — 서울시청 폴백 표시, 설정 이동 알럿

## 1. 카메라 점프 제거

- [ ] 첫 실행 + 권한 허용: 서울시청 표시 → 허용 후 현재 위치로 부드럽게 이동
- [ ] 첫 실행 + 권한 거부: 서울시청에 그대로 머묾 + 설정 이동 알럿
- [ ] 재실행 (이미 사용한 적 있음): 마지막으로 본 위치가 바로 보임, 자동 이동 없음
- [ ] "내 위치" 버튼 탭: 현재 위치로 부드럽게 이동

## 2. 비연속 구간 그리기

- [ ] 기존 경로 끝점 근처에서 새로 그리기 → 뒤에 자연스럽게 연장
- [ ] 기존 경로 시작점 근처에서 새로 그리기 → 앞에 자연스럽게 연장
- [ ] 기존 경로와 떨어진 곳에서 그리기 → 가까운 쪽(시작/끝)에 연결, 역주행 없음
- [ ] 되돌리기 → 마지막으로 추가한 구간만 제거, 나머지 경로 유지
- [ ] 되돌리기 여러 번 → 순서대로 제거, 빈 상태까지 가능
- [ ] 의도와 다른 방향으로 연결되는 케이스가 있는지 메모

## 3. 스로틀 완화 (증분 계산)

- [ ] 스트로크 3~4개 연속 그리기 → 매끄럽게 경로 표시, 이전보다 빠른 체감
- [ ] 스트로크 7~10개 빠르게 연속 그리기 → 스로틀 에러 발생 빈도 이전보다 감소했는지
- [ ] 스로틀 에러 발생 시 Xcode 콘솔에 `[MapKitCoursePlanning]` 로그 확인 → domain/code 메모
- [ ] 스로틀 에러 후 잠시 대기 → 다시 그리기 정상 동작

## 제스처/입력 감
- [ ] 탭 모드 ↔ 그리기 모드 전환 자연스러움
- [ ] 초기화 버튼 동작
- [ ] 전체적인 터치 반응 자연스러움

## 엣지/견고성
- [ ] 빠른 모드 전환 반복 (탭↔그리기) → 크래시 없음
- [ ] 빠른 되돌리기 반복 → 크래시 없음
- [ ] 그리기 중 앱 백그라운드/포그라운드 → 상태 유지

## 의도 일치
