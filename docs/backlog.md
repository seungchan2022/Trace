# Backlog

실기기 테스트·리뷰에서 나온 개선/수정 후보와 미룬 기술부채를 모으는 곳.
**규칙:** backlog 항목은 **마일스톤 후보**다 — 묶어서 새 MVP를 구성하거나 기존 MVP의 마일스톤으로 편입한다(작고 명확하면 spec/plan 바로, 결정·모호하면 brainstorm). 큐가 아니라 **메뉴** — 새 기능을 먼저 해도 된다. 단위·흐름은 `docs/agent-rules/workflow.md`, 현재 위치는 `docs/roadmap.md`, 캡처/소비 흐름은 `testing.md`·`skills.md` 참고.
상태: `open`(미착수) / `planned`(마일스톤 잡힘) / `done`(완료).

## 마커 그리기 + 스냅 MVP (2026-06-20) 실기기 피드백

- [ ] **위치 시작 화면** — *where:* 진입 시 지도 / *now:* 현재 위치로 대략 줌인만 됨 / *desired:* 더 줌인 + 현재 위치에 핀(점) 표시 + "내 위치로" 재중심 버튼. `open`
- [ ] **그리기 모드 표시 명확화** — *where:* 그리기 토글 / *now:* 라벨이 "그리기 종료"일 때가 모드 ON인데 헷갈림 / *desired:* 모드 ON 상태가 직관적으로 보이는 어포던스(별도 인디케이터/색/상태표시 등). `open`
- [ ] **그린 코스에 출발/도착 핀** — *where:* 드래그로 만든 경로 / *now:* 핀이 탭 방식에서만 표시 / *desired:* 그린 경로의 시작·끝에도 출발/도착 핀 표시. `open`
- [ ] **상태 잔상 정리 + 상호작용 모델** *(결정 필요)* — *where:* 탭-포인트와 그리기 공존 / *now:* 두 점 찍고 드래그/초기화해도 옛 출발·도착이 남음 / *desired:* 초기화 = 모든 표시 제거, 두 점 찍은 뒤 새 드래그 시작 시 옛 포인트도 정리. *→ 탭 방식과 그리기 방식을 어떻게 공존/통합할지 brainstorm 필요.* `open`
- [ ] **권한 거부 UX** *(제품 결정)* — *where:* 위치 권한 거부 시 / *now:* 조용히 서울시청으로 폴백 / *desired:* 기능 잠금 + "설정에서 위치 권한 켜기" 안내(설정 이동). *→ '위치 없이도 쓰게 둘지'를 정하는 결정.* `open`
