# MVP16 ui-direction — 디자인 결정 (마일스톤 1)

- **작성일:** 2026-07-19
- **상태:** 인터뷰 확정 (경량 사이클 — 문서 전용, 코드 없음. MVP12 design-direction 방식)
- **상위 문서:** `2026-07-19-mvp16-ui-restructure-kickoff-design.md`(결정 레이어) · `2026-07-18-run-ui-restructure-direction.md`(방향)
- **참고 캡처:** `docs/refs/` — 나의찾기 다크 시트 2장, 네이버 지도 시트-탭바 구조 3장(collapsed/medium/full)
- **원칙:** 뼈대(레이아웃 위계)는 장르 표준을 따르고, 살(색·폰트·컴포넌트)은 Trace 디자인 토큰을 입힌다 — "나이키식 구성의 Trace 화면"이지 카피가 아니다. 모든 결정은 실기기 확인 후 수정 가능한 기본값.

---

## 1. 커스텀 탭바

- **형태:** 풀폭 불투명 바. 탭 2개(코스/러닝), 아이콘 + 한글 라벨. 선택 탭은 민트 액센트, 비선택은 보조 텍스트 색. 배경은 시트와 동일한 Surface 토큰(위쪽 경계는 헤어라인).
- **노출 규칙:**
  - 코스 탭: 시트 디텐트와 무관하게 **상시 노출** — 시트는 탭바 *위에* 얹히고, 풀 디텐트에서도 탭바는 보인다(네이버 지도 문법, `docs/refs/naver-map-sheet-full.png`).
  - 러닝 탭: 대기 화면에서만 노출. 러닝 시작 → 요약 화면을 닫을 때까지 숨김(킥오프 §2.2).
- **전환 트랜지션:** 없음(즉시 전환)이 기본값. 밋밋하면 크로스페이드 정도만 검토 — 슬라이드 등 방향성 애니메이션은 두 탭뿐이라 과함.
- **냉시작 기본 탭:** 코스 (첫 기둥, 현행 유지).
- **시스템 탭 badge(●) 제거:** 현재 `TraceApp`의 러닝 세션 badge는 "트래킹 중 다른 탭에 있을 때" 알림용인데, 트래킹 중 탭 전환 자체가 사라지므로 죽은 기능 — 커스텀 탭바에 이식하지 않는다.

## 2. 코스 탭 (구조 요약 — 상세는 킥오프 스펙)

- 기존 커스텀 시트(`SheetDetent` collapsed/medium/full) 유지, 시스템 `.sheet` 불사용.
- 시트가 커스텀 탭바 위에 얹히는 3층 구조: 지도(풀블리드) → 시트 → 탭바.
- 플로팅 버튼(우상단): 경로 없음 = 현위치만 / 경로 있음 = 전체 등장, 시트 상승에 따라 이동·페이드, 풀 디텐트에서 소멸(방향 스펙 그대로).
- detent는 "전환"이 아니라 현행 비율 기반 검증·보완(문서 리뷰 정정 반영).

## 3. 러닝 트래킹 화면 — "거리가 주인공"

지도 없는 숫자 중심 전체화면. NRC 등 러닝앱 장르 표준 위계 + Trace 토큰.

```
┌─────────────────────────┐
│  (상태바)                │
│                         │
│      12'34"   5'42"     │  ← 보조 행 위: 시간 · 평균 페이스 (중형)
│                         │
│        3.42             │  ← 누적 거리 (초대형, 화면 주인공)
│         km              │
│                         │
│   ▓▓▓▓▓▓░░░░  68%       │  ← 목표 진행률 (목표 러닝일 때만)
│                         │
│                         │
│   ┌──────┐  ┌──────┐    │
│   │ 포인트│  │일시정지│   │  ← 컨트롤 영역
│   └──────┘  └──────┘    │     종료 = 기존 홀드 링 (일시정지 중 노출)
│                         │
│      (탭바 없음)         │
└─────────────────────────┘
```

- 트래킹 중 조작은 일시정지/종료/포인트 셋(킥오프 §2.2). 포인트 버튼은 컨트롤 영역에 일시정지와 나란히 — 즉시 발화·잠금화면 버튼 등 기존 동작 무변경.
- 일시정지 상태 표기·종료 홀드 링·GPS 신호 확보 중 전이는 기존 동작 유지, 배치만 이 레이아웃을 따른다.
- 다크/라이트 모두 Surface 토큰. 숫자는 SF 모노스페이스드 숫자(기존 스탯 패널과 동일 계열)로 자릿수 흔들림 방지.

## 4. 러닝 대기 화면 — 시작 중심 미니멀

지도 배경 제거(킥오프 §2.3 — 러닝 탭 전체). Surface 배경 위:

```
┌─────────────────────────┐
│              [기록 ▸]   │  ← 우상단 기록 목록 진입 (유지)
│                         │
│   자유 · 거리 · 시간      │  ← 목표 모드 선택 (직전값 프리필 유지)
│      [ 5.0 km ]         │  ← 선택 모드의 값 입력 (거리/시간일 때)
│                         │
│                         │
│       ┌───────┐         │
│       │  시작  │         │  ← 대형 시작 버튼 (민트, 화면 주인공)
│       └───────┘         │
│                         │
│  코스 │ 러닝              │  ← 커스텀 탭바 (대기 화면에서만 노출)
└─────────────────────────┘
```

- 목표 설정 UI(자유/거리/시간, 단위 상시 표시, 인라인 에러)는 MVP15 동작 그대로 — 배치만 재구성.
- 최근 기록 카드는 넣지 않는다(기록 목록과 역할 중복, 초점 분산). 기록 탭 승격 재검토 시 함께 재론.

## 5. 요약 화면 · 기록 화면

- **요약 화면:** 탭바 없는 전체화면 유지(킥오프 §2.2 — 닫아야 대기 화면+탭바 복귀). 콘텐츠 구성(최종 수치·스플릿)은 현행 유지, 스타일만 토큰 정리.
- **기록 목록/상세:** 러닝 탭 안 전체화면 페이지 문법으로 정리(킥오프 §2.1). 탭바 노출 여부는 **대기 화면과 동일 계층이므로 노출** — 러닝 플로우(시작~요약)만 탭바를 숨긴다.

## 6. 기록 상세 지도 — 포인트 구간별 색상 폴리라인

- 경로를 포인트 경계에서 잘라 **구간마다 다른 색**으로 렌더링. 색은 코스 플래너의 `SegmentPalette` 문법 재사용 — 기록 상세의 포인트 구간 표와 지도가 색으로 대응된다(플래너 구간 리스트↔지도와 동일한 언어).
- 포인트 없는 기록은 현행 단일색 유지. 번호 마커는 유지(색상 폴리라인과 병행).
- **왕복 겹침은 일단 무대응** — 표시 전용이라 겹치면 나중에 그린 구간이 위에 보이는 것으로 시작. *트리거:* 실사용에서 겹침 구간 판독 불편이 확인되면 플래너의 오프셋 렌더링(`OverlapOffsetResolver`) 이식 검토.
