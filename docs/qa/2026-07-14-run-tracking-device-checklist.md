# 자유 러닝 실기기 체크리스트 (2026-07-14)

> 코스를 짜지 않고 그냥 뛰어도 시작 → 뛰는 동안 기록 → 종료 요약이 되는 새 화면입니다.
> 시뮬레이터로는 위치 신호를 흉내만 낼 수 있어서, 실제로 들고 뛰면서 확인해야 하는
> 부분(진짜 위치 신호, 화면을 잠근 채로도 계속 기록되는지, 배터리 체감)이 많습니다. 아래
> 체크리스트를 실제로 밖에 나가 짧게(10~15분 정도) 뛰어보면서 채워 주세요.

빌드/테스트는 전부 통과했습니다(자동 테스트 전체 통과, 회귀 없음). 시뮬레이터에서 가짜
위치 신호로 흉내 내 확인했을 때는 정상이었지만, 실제 위치 신호·화면 잠금 중 기록·배터리
체감은 시뮬레이터로는 확인이 안 되는 부분이라 아래 항목들을 실제 기기에서 봐주시면
좋겠습니다.

**사용법:** 위에서부터 순서대로 하나씩 수행한다. 수행 후 실제로 본 결과가 기대와 같으면
통과에 체크, 다르면 실패에 체크하고 메모에 무슨 일이 일어났는지 짧게 적는다.

## 빌드/설치
- [ ] 기기 연결, 자동 서명 팀 확인, Xcode Run 성공
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰(설정 > 일반 > VPN 및 기기 관리)

## 권한
- [ ] 앱을 처음 열고 러닝 화면으로 가서 "시작"을 눌렀을 때 뜨는 실제 위치 허용 팝업의
      문구를 확인하고, "앱을 사용하는 동안 허용"을 선택해 정상 진행되는지 확인
- [ ] (한 번 더 지워보고) 이번엔 위치 요청을 거부해 봤을 때 앱이 멈추거나 죽지 않고,
      "위치 권한이 필요해요" 같은 안내와 설정으로 이동하는 버튼이 뜨는지 확인

## 핵심 기능 (손으로 수행)

### 시나리오 1: 뛰기 시작 전, 위치를 잡는 동안의 화면

**준비:** 위치 권한을 이미 허용한 상태. 실외(하늘이 보이는 곳)에서 시작한다.

**수행:**
1. 러닝 화면에서 가운데 동그란 "시작" 버튼을 누른다.
2. 화면 아래쪽에 뜨는 안내를 지켜본다.

**확인할 것:** 위치를 찾는 동안 "GPS 신호 찾는 중…" 같은 문구와 함께 빙글빙글 도는
표시가 나오고, 실제로 몇 초 안에 신호를 잡아 다음 화면(거리·시간·페이스가 보이는 화면)으로
자동으로 넘어가야 한다. 이 상태에서 "취소"를 누르면 원래 시작 화면으로 깔끔하게
돌아가야 한다(취소는 따로 한 번 눌러서 확인).

**결과:** ☐ 통과 ☐ 실패

### 시나리오 2: 뛰는 동안 거리·시간·페이스·지나온 길이 제대로 나오는지

**준비:** 시나리오 1을 마치고 기록이 시작된 상태.

**수행:**
1. 실제로 1~2분 정도 걷거나 가볍게 뛴다.
2. 화면 아래쪽 숫자들(거리, 경과 시간, 페이스)이 바뀌는지 지켜본다.
3. 화면 위쪽 지도에 방금 지나온 길이 선으로 그려지는지 본다.

**확인할 것:** 걷거나 뛴 만큼 거리 숫자가 늘어나야 하고, 시간은 초 단위로 계속
올라가야 한다. 페이스(속도) 숫자도 실제로 걷는 속도에 맞게 나와야 한다(가만히 서
있으면 페이스가 터무니없이 크게 나오는 것은 정상). 지도 위 선이 실제로 걸은 길
모양을 따라가야 한다.

**결과:** ☐ 통과 ☐ 실패

### 시나리오 3: 화면을 끄거나 잠가도 잠금 화면에 기록이 계속 나오는지

**준비:** 뛰는 중(시나리오 2 상태)이어야 한다.

**수행:**
1. 기기 옆면 버튼을 눌러 화면을 끈다(잠근다).
2. 몇 초 뒤 다시 옆면 버튼을 눌러 잠금 화면을 켠다(밀어서 열지는 않는다).
3. 잠금 화면 위쪽이나 알림 자리에 나오는 러닝 카드를 본다.
4. 잠근 채로 20~30초 정도 그대로 두었다가 다시 잠금 화면을 켜서 숫자가 바뀌었는지
   비교한다.

**확인할 것:** 잠금 화면에 뛰는 사람 아이콘과 함께 거리·시간·페이스가 보이는 카드가
나와야 한다. 화면을 잠근 동안에도 시간은 계속 흐르고, 다시 잠금 화면을 켰을 때 거리
숫자도 그사이 이동한 만큼 늘어나 있어야 한다(멈춰 있지 않아야 함).

**결과:** ☐ 통과 ☐ 실패

### 시나리오 4: 다른 앱에 갔다가 돌아와도 기록이 계속되는지

**준비:** 뛰는 중이어야 한다.

**수행:**
1. 홈 화면으로 나가서 지도 앱이나 음악 앱 등 다른 앱을 열어 10~20초 정도 있는다.
2. 다시 이 앱으로 돌아온다.

**확인할 것:** 다른 앱에 가 있는 동안에도 기록이 멈추지 않아야 한다. 앱으로 돌아왔을 때
거리·시간이 나가 있던 시간만큼 늘어나 있어야 하고, 지나온 길도 끊기지 않고 이어져
있어야 한다. 처음부터 다시 시작된 것처럼 보이면 안 된다.

**결과:** ☐ 통과 ☐ 실패

### 시나리오 5: 코스 화면과 러닝 화면을 오갈 때 표시

**준비:** 뛰는 중이어야 한다.

**수행:**
1. 화면 아래쪽 탭에서 "코스" 탭을 눌러 지도 화면으로 이동한다.
2. "코스" 탭 옆에 있는 "러닝" 탭에 작은 점 같은 표시가 있는지 본다.
3. 다시 "러닝" 탭을 눌러 돌아온다.

**확인할 것:** 뛰는 중에는 "러닝" 탭에 작은 초록색 점 같은 표시가 붙어 있어야
한다(지금 기록 중이라는 표시). "코스" 탭에서 코스 화면을 보다가 "러닝" 탭으로
돌아오면 방금 하던 기록 화면(거리·시간·페이스)이 그대로 남아 있어야 한다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

### 시나리오 6: 다 뛰고 나온 숫자가 말이 되는지

**준비:** 시나리오 2~5를 거쳐 어느 정도(5분 이상) 뛴 다음, 아래 "제스처/입력 감"의
종료 방법으로 기록을 마친 상태.

**수행:**
1. 종료 후 나오는 요약 화면의 거리·시간·평균 페이스·고도(오르내림) 숫자를 본다.
2. 실제로 뛴 거리·시간과 비교해 본다.
3. 특히 평평한 길에서만 뛰었다면 고도 숫자를 유심히 본다.

**확인할 것:** 거리와 시간은 실제로 뛴 만큼과 크게 다르지 않아야 한다. 평평한 길에서만
뛰었다면 고도(오르내림) 숫자가 0에 가깝거나 아주 작아야 한다 — 언덕 하나 없었는데
수십 미터씩 올라간 것처럼 나오면 이상한 것이다.

**결과:** ☐ 통과 ☐ 실패

## 제스처/입력 감

### 시나리오 7: 길게 눌러야만 종료되는 버튼

**준비:** 뛰는 중이어야 한다.

**수행:**
1. 화면 아래쪽 빨간 "길게 눌러 종료" 버튼을 손가락으로 아주 짧게 톡 눌렀다 뗀다.
2. 이번엔 손가락을 대고 1초 이상 꾹 누르고 있는다.

**확인할 것:** 1번처럼 짧게 톡 누르는 것만으로는 기록이 끝나면 안 된다(실수로 뛰다가
스친 정도로는 종료되지 않아야 함). 2번처럼 꾹 누르고 있어야만 요약 화면으로 넘어가야
한다. 누르고 있는 동안 버튼이 살짝 눌리는 느낌(크기 변화)이 있어 지금 누르는 중이라는
것이 손가락으로도 느껴져야 한다.

**결과:** ☐ 통과 ☐ 실패

## 지역/환경 의존

### 시나리오 8: 건물 사이 vs 하늘이 트인 곳에서 위치 정확도 차이

**준비:** 뛰기 전.

**수행:**
1. 높은 건물 사이나 실내 근처처럼 하늘이 잘 안 보이는 곳에서 한 번 시작해 본다.
2. 공원처럼 하늘이 트인 곳에서 다시 한번 시작해 본다.

**확인할 것:** 건물 사이에서는 위치를 잡는 데 시간이 더 걸리거나, 지도 위 선이 실제
걸은 길과 살짝 어긋나 보일 수 있다(있을 수 있는 일이고 고장이 아니다). 다만 신호가
약할 때는 화면에 "GPS 신호 약함" 같은 안내가 떠야 하고, 트인 곳에서는 이런 안내 없이
선이 실제 걸은 길과 잘 맞아야 한다.

**결과:** ☐ 통과 ☐ 실패

## 엣지/견고성

### 시나리오 9: 개발자용 기록 파일을 꺼내 보기

**준비:** 시나리오 6에서 본 요약 화면이 떠 있는 상태(테스트 버전에서만 보이는 화면).

**수행:**
1. 요약 화면에 "샘플 덤프 내보내기 (DEBUG)"라고 적힌 버튼이 있으면 눌러 본다.
2. 아이폰의 공유하기 창(메시지·에어드롭·파일에 저장 등이 뜨는 창)이 열리는지 본다.
3. "파일에 저장"을 눌러 아무 곳에나 저장해 본다.

**확인할 것:** 버튼을 누르면 바로 공유하기 창이 뜨고, 파일로 저장하면 정상적으로
저장이 끝나야 한다(에러 창이 뜨거나 앱이 멈추면 안 됨). 이 버튼은 테스트해 주시는
버전에서만 보이는 개발자용 기능이라, 나중에 정식으로 나오는 앱에는 없어도 정상이다.

**결과:** ☐ 통과 ☐ 실패
**메모:**

### 시나리오 10: 뛰는 도중 앱을 완전히 꺼도 잠금 화면 카드가 같이 사라지는지

**준비:** 뛰는 중이어야 한다.

**수행:**
1. 화면을 잠그지 않은 채로, 홈 버튼을 위로 쓸어올리거나 눌러서 최근 사용한 앱 목록(앱
   스위처)을 연다.
2. 이 앱 카드를 위로 쓸어올려서 앱을 완전히 종료한다(잠깐 나갔다 오는 게 아니라 아예
   꺼버리는 것).
3. 몇 초 기다린 다음, 옆면 버튼으로 화면을 잠그고 다시 켜서 잠금 화면을 본다.
4. 이 앱을 다시 실행한다.

**확인할 것:** 앱을 완전히 꺼도 잠금 화면에 러닝 카드가 계속 남아 있으면 안 된다 —
화면을 껐다 켰을 때 카드가 사라져 있어야 한다(숫자가 멈춘 채로 계속 떠 있으면 실패).
앱을 다시 열면 뛰던 기록은 남아 있지 않고 처음 시작 화면부터 다시 시작해야 한다(이건
정상 동작이다 — 완전 종료 시 기록이 사라지는 건 알려진 한계).

**결과:** ☐ 통과 ☐ 실패
**메모:**

- [ ] **배터리 소모 체감:** 뛰기 시작 전 배터리 잔량(%)을 확인해 두고, 뛰기를 마친
      뒤 다시 배터리 잔량을 확인한다. 뛴 시간에 비해 배터리가 유난히 빨리 닳는다는
      느낌이 드는지 메모(정확한 숫자보다 "이 정도면 괜찮다/너무 빨리 닳는다" 체감
      위주로 적어 주세요).
      **메모:**

## 의도 일치

      비교했을 때 어색하거나 불편하게 느껴지는 부분이 있는지(느낌 위주로 자유롭게)

**메모:**

## 알려진 한계 (버그 아님)
- [ ] 코스 화면에서 짠 경로와 러닝 기록은 아직 서로 연결되어 있지 않다(코스를 골라
      그대로 따라 뛰는 기능은 다음에 추가될 예정)
- [ ] 뛴 기록은 저장되지 않는다 — 앱을 완전히 종료하면 방금 뛴 기록도 사라진다(기록을
      남겨서 목록으로 보는 기능은 다음 업데이트 예정)
- [ ] 심박수 등 몸 상태는 측정하지 않는다(위치로 잰 거리·시간·속도·고도만 기록)
- [ ] "샘플 덤프 내보내기 (DEBUG)" 버튼은 테스트 버전에서만 보이는 개발자용 기능이다
