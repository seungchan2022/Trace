# MVP8 코스 편집·가독성 UX 실기기 체크리스트 (2026-07-03)

> 마일스톤 1(course-ux-polish: 구간 패널 스크롤/높이 제한 + 핀치 줌 네이티브 복원)과
> 마일스톤 2(overlap-offset: 겹치는 경로 좌표 오프셋 렌더링)를 묶어 한 번에 제출한다.
> 템플릿: `docs/agent-rules/testing.md`

## 빌드/설치

- [ ] 기기 연결, 자동 서명 팀 확인, Xcode Run 성공
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰(설정 > 일반 > VPN 및 기기 관리)

## 마일스톤 1 — 구간 패널 + 핀치 줌

- [ ] 구간 10개 이상 만들었을 때 패널이 화면(지도 높이) ~40%에서 멈추고 그 안에서 스크롤되는지
- [ ] 최신 구간 근처를 보고 있을 때 새 구간을 추가하면 목록이 계속 최신을 따라가는지 (prepend로 붙는 경우 포함)
- [ ] 옛 구간을 스크롤해서 보는 중에 새 구간이 추가되면 보던 위치가 유지되는지 (자동으로 안 끌려가는지)
- [ ] 패널을 접었다 다시 펼쳤을 때 이전에 보던 위치로 복원되는지 (최신 근처였다면 최신으로)
- [ ] **화면이 큰 기기(Pro Max급)에서 40% 상한 안에 짧은 행이 7~8개 이상 보일 때, 최신 구간을 보며 구간을 연속으로 여러 개 추가해도 매번 계속 최신을 따라가는지** (한 번만 따라가고 멈추지 않는지 — 2026-07-02 브랜치 리뷰에서 발견된 앵커 불일치 가능성, 재현되면 `docs/superpowers/plans/2026-07-02-course-ux-polish.md` Task 4 참고)
- [ ] 그리기 모드에서 두 손가락 핀치가 손가락 사이 지점을 기준으로 자연스럽게 줌되는지 (기존 수제 줌 대비 체감 개선 여부)
- [ ] 더블탭 줌이 그리기 시작과 간섭하지 않는지
- [ ] 두 손가락 탭 줌아웃이 커스텀 2손가락 pan과 간섭하지 않는지

## 마일스톤 2 — 겹치는 경로 오프셋

- [ ] 왕복 코스(같은 길을 되돌아오는 코스)에서 두 선이 나란히 보이고, 각 거리 라벨이 자기 선 위에 있는지
- [ ] 왕복 코스에서 도착 핀과 선 끝이 일치하는지 (핀 지점은 오프셋 예외)
- [ ] 코스 전체를 조망하는 축소 배율에서 라벨-선 색 어긋남이 재현되는지 (재현되면 화면 포인트 오프셋 방식 백로그 트리거 — `docs/backlog.md` MVP8 킥오프 논의 참고)
- [ ] 줌 배율을 바꿔가며 두 선이 한 줄로 합쳐 보이기 시작하는 시점 기록, 현재 10m 감지/4m 오프셋 초기값이 체감상 적절한지
- [ ] **짧은 겹침 구간(왕복 구간이 테이퍼 길이 15m보다 짧게 겹치는 경우)에서 목표 오프셋(4m)까지 도달하지 못하고 상대적으로 덜 벌어져 보이는지** (2026-07-02 Task 3 리뷰에서 확인된 알고리즘 특성 — 수학적으로 의도된 동작이지만 실사용 체감 확인 필요)
- [ ] 겹치는 스트로크가 확정되는 순간 위치가 바로 튀는(즉시 스냅) 것이 거슬리는지 (거슬리면 애니메이션 백로그 등록)
- [ ] 좁은 골목·도로변 산책로에서 실제로 안 겹쳤는데 밀려 보이는 거짓 양성이 있는지
- [ ] 구간을 추가(attach)한 직후 프레임 히치(끊김)가 느껴지는지 (겹침 감지 비용 실측 — 문제 시 격자 해싱 최적화 백로그)
