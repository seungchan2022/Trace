# MVP10 gesture-consistency 실기기 체크리스트 (2026-07-04)

> 탭 즉시 확정을 "0.3초 보류 → 확정/취소"로 교체해 더블탭·원핑거 줌과 분리하고,
> 그리기 시작점 근접 판정을 화면 픽셀(24pt) 기준 + 실거리 20m 폴백으로 바꾸고,
> 그리기 모드 두손가락 제스처(탭줌아웃 ↔ 커스텀 팬) 경쟁을 튜닝한다.
> 스펙: `docs/superpowers/specs/2026-07-04-gesture-consistency-design.md`
> 플랜: `docs/superpowers/plans/2026-07-04-gesture-consistency.md`
> 템플릿: `docs/agent-rules/testing.md`

빌드/테스트/린트 전부 통과, 시뮬레이터 스모크(탭 2회 경로 생성, 다른 위치 빠른 연속 탭 2회,
그리기 1회, undo)에서 크래시·유령 마커 없음 확인. 아래는 시뮬레이터·유닛 테스트로는
검증할 수 없는 항목이다(더블탭/원핑거 줌 구분, 실제 손가락 제스처 경쟁, 실거리 판정 체감).

## 빌드/설치

- [ ] 기기 연결, 자동 서명 팀 확인, Xcode Run 성공
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰(설정 > 일반 > VPN 및 기기 관리)

## 핵심 기능 (손으로 수행)

- [ ] 싱글탭: 마커 즉시 표시 → 경로 계산 (체감 지연 없음)
- [ ] 더블탭: 줌인만 — 포인트 안 생김, 임시 마커 깜빡임 수용 가능한지 메모
- [ ] 탭 후 두 번째 터치 유지 상하 드래그: 원핑거 줌만 — 포인트 안 생김
- [ ] 경계 간격(~0.3–0.4초) 더블탭: 줌+포인트 동시 발생 없음
- [ ] 서로 다른 위치 빠른 연속 탭: 둘 다 포인트
- [ ] 출발핀 탭 왕복: 기존과 동일 동작
- [ ] 판별 창 중 모드 전환: 잔류 마커 없음
- [ ] 줌아웃 상태에서 핀 옆(화면상)에서 그리기 시작: 이어붙음 (51m 케이스 해결 확인)
- [ ] 고배율 줌인에서 핀 근처(실거리 20m 이내) 그리기 시작: 여전히 이어붙음 (폴백 확인)
- [ ] 두손가락 탭줌아웃 ↔ 두손가락 팬 빠른 연속: 각각 정상 동작

## 제스처/입력 감

- [ ] 탭 보류(~0.35초) 동안 체감 지연이 거슬리지 않는지 (정상 단일 탭 흐름 기준)
- [ ] 빠른 연속 탭/그리기 전환에서 판별기 상태가 씹히거나 중복 실행되지 않는지

## 지역/환경 의존

- [ ] 한국 도로망/실제 지도 확대 배율에서 24pt 픽셀 판정 임계값이 체감상 적절한지 (너무 좁아서 안 잡히거나, 너무 넓어서 의도치 않게 이어붙는지)

## 엣지/견고성

- [ ] 판별 창(~0.35초) 중 앱을 백그라운드로 전환했다가 복귀해도 잔류 마커나 크래시 없는지
- [ ] 그리기 모드에서 두손가락 제스처를 반복적으로 빠르게 전환해도 안정적인지

## 의도 일치

- [ ] 만든 기능이 원한 UX와 맞는가 (불일치 메모)

## 알려진 한계 (버그 아님)

- [ ] 시뮬레이터에서는 더블탭/원핑거 줌 취소 경로, 두손가락 제스처 경쟁을 안정적으로 재현할 수 없어 대부분 코드 리뷰 + 유닛 테스트(TapClassifier 상태 머신)로만 검증됨 — 이 체크리스트가 해당 경로의 첫 실기기 검증
