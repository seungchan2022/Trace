# 코스 저장 + 구간 왕복 실기기 체크리스트 (2026-07-07)

> 지도에서 그린 코스가 앱을 껐다 켜도 그대로 남아있게(자동 저장·복원) 하고, 완성한 코스에
> 이름을 붙여 저장했다가 나중에 목록에서 골라 다시 불러올 수 있게 하고, 구간 하나를 골라
> "왕복"(갔다가 그대로 되돌아오기)으로 바꿀 수 있게 하는 기능입니다.

자동으로 돌리는 검증은 모두 통과했고, 화면으로 한 번 실행해서(코스 생성 → 왕복 추가 →
이름 저장 → 초기화 → 목록에서 불러오기 → 앱 완전 종료 후 재실행 → 복원 확인) 문제없이
동작하는 것도 확인했습니다. 아래는 화면 실행만으로는 체감하기 어려운 항목이다(실제 손가락으로 앱을 껐다 켜는 느낌,
알림창 문구가 실제로 이해되는지, 진짜로 다시 실행해도 안 지워지는지에 대한 안심감).

**사용법:** 위에서부터 순서대로 하나씩 수행한다. 각 시나리오는 "준비 → 수행 → 확인할 것"
순으로 읽고, 수행 후 실제로 본 결과가 기대와 같으면 통과에 체크, 다르면 실패에 체크하고
메모에 무슨 일이 일어났는지 짧게 적는다. 실패 항목은 나중에 한 번에 모아서 알려주면 된다.

## 빌드/설치

- [ ] 기기 연결, 자동 서명 팀 확인, Xcode Run 성공
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰(설정 > 일반 > VPN 및 기기 관리)

## 핵심 기능 (손으로 수행)

### 시나리오 1: 앱을 완전히 껐다 켜도 그리던 코스가 그대로 남아있는지

**준비:** 앱을 열고, 지도를 몇 번 탭해서 코스를 하나 그린다(점을 3~4개 정도 찍어서 길이 생기게).

**수행:**
1. 코스가 그려진 상태에서, 앱을 최근 앱 목록에서 위로 쓸어올려 완전히 종료한다(그냥 홈 화면으로
   나가는 것이 아니라 진짜로 종료해야 한다).
2. 앱을 다시 실행한다.
3. 방금 그렸던 코스가 지도에 그대로 남아있는지 본다.
4. "되돌리기" 버튼을 한 번 눌러본다.

**확인할 것:** 껐다 켜기 전과 똑같은 길이 그대로 보여야 한다(점 개수, 거리 숫자 모두 동일).
"되돌리기"를 누르면 방금 찍은 점 하나가 정상적으로 취소되어야 한다(즉 껐다 켠 뒤에도
되돌리기 기능이 계속 잘 동작해야 한다).

**결과:** ☐ 통과 ☐ 실패

### 시나리오 2: 코스를 지우고 종료하면, 다시 열었을 때 빈 상태로 시작하는지

**준비:** 코스가 그려진 상태(시나리오 1 이어서 해도 됨).

**수행:**
1. "초기화" 버튼을 눌러 코스를 지운다.
2. 지도가 빈 상태(거리 0km)가 된 것을 확인한다.
3. 앱을 완전히 종료한다.
4. 앱을 다시 실행한다.

**확인할 것:** 다시 열었을 때 아까 지웠던 코스가 되살아나 있으면 안 된다. 빈 지도, 0km
상태로 시작해야 한다.

**결과:** ☐ 통과 ☐ 실패

### 시나리오 3: 이름 붙여 저장 → 목록에서 확인 → 다른 코스 작업 중 불러오면 확인 창이 뜨는지 → 삭제도 되는지

**준비:** 코스를 하나 그린다.

**수행:**
1. 저장 버튼(아래 화살표 아이콘)을 눌러 이름을 입력하고 저장한다(예: "테스트1").
2. 목록 버튼(줄 아이콘)을 눌러 저장된 코스 목록을 연다. 방금 저장한 이름이 목록에 보이는지 확인한다.
3. 목록을 닫고, "초기화" 없이 곧바로 지도를 몇 번 더 탭해서 새로운 코스를 그린다(방금 저장한
   것과 다른, 아직 저장 안 된 코스).
4. 다시 목록을 열어 아까 저장한 "테스트1"을 탭한다.
5. 화면에 뜨는 확인 문구를 읽는다.
6. 확인을 눌러 실제로 바뀌는지 본다.
7. 목록을 다시 열어, 방금 불러온 코스를 왼쪽으로 밀어(스와이프) 삭제해본다.

**확인할 것:** 4번 단계에서 "지금 작업 중인 코스를 버리고 다른 코스로 바꿀지" 묻는 확인
창이 떠야 한다(그냥 조용히 바뀌면 안 됨). 확인을 누르면 지도가 진짜로 "테스트1" 코스로
바뀌어야 한다. 7번에서 삭제를 확정하면 목록에서 그 항목이 실제로 사라져야 한다.

**결과:** ☐ 통과 ☐ 실패

### 시나리오 4: 구간 하나를 왕복으로 바꾸면 그 구간 거리가 두 배가 되는지, 되돌리기로 취소되는지

**준비:** 점을 3개 이상 찍어서 구간이 2개 이상 있는 코스를 그린다. 화면 오른쪽 위의 거리
칩(예: "0.6 km")을 눌러 구간 목록 패널을 펼친다.

**수행:**
1. 맨 위(또는 중간) 구간 옆에 있는 왕복 버튼을 누른다.
2. 그 구간의 거리 숫자가 어떻게 바뀌는지 본다.
3. 전체 거리(위쪽 칩 숫자)가 어떻게 바뀌는지 본다.
4. "되돌리기" 버튼을 한 번 누른다.

**확인할 것:** 왕복 버튼을 누르면 그 구간의 거리가 정확히 두 배로 늘어나야 하고, 지도에도
그 자리에 갔다가 되돌아오는 선이 겹쳐서 보여야 한다. "되돌리기"를 한 번 누르면 왕복을
누르기 전 상태로 완전히 돌아가야 한다(구간 개수, 거리 모두 원래대로).

**결과:** ☐ 통과 ☐ 실패
**메모:**

### 시나리오 5: 왕복이 포함된 코스를 저장했다가 불러와도 왕복 표시가 남아있는지

**준비:** 시나리오 4에서 만든, 왕복이 포함된 코스(되돌리기를 누르지 않은 상태).

**수행:**
1. 저장 버튼을 눌러 이름을 붙여 저장한다(예: "왕복테스트").
2. "초기화"로 지도를 비운다.
3. 목록을 열어 "왕복테스트"를 눌러 불러온다.
4. 구간 목록 패널을 펼쳐서 왕복을 걸었던 구간을 찾아본다.

**확인할 것:** 불러온 뒤에도 그 구간에 "왕복 구간" 표시(화살표 아이콘)가 그대로 남아있어야
하고, 거리 숫자도 저장했을 때와 동일해야 한다.

**결과:** ☐ 통과 ☐ 실패
