# 러닝 일시정지/재개 실기기 체크리스트 (2026-07-16)

> 러닝 중에 잠깐 멈췄다가 다시 이어서 뛸 수 있는 기능입니다. 신호 대기, 잠깐 쉬기 같은
> 상황에서 시간과 거리가 실제로 멈추는지 확인합니다. 짧게(1~2분) 걷거나 제자리에서
> 움직이는 정도로 충분합니다. 시뮬레이터로는 "화면을 끄고 주머니에 넣은 채 잠금화면
> 카드가 일시정지 상태로 유지되는지"를 확인할 수 없어서 아래 항목들을 실제 기기에서
> 봐주시면 좋겠습니다.

빌드/테스트는 전부 통과했습니다(자동 테스트 236개 + UI 테스트 6개 전체 통과, 회귀 없음).
시뮬레이터에서는 일시정지/재개 버튼 조작과 타이머 멈춤·재개 표시를 눈으로 확인했습니다.

**사용법:** 위에서부터 순서대로 하나씩 수행한다. 수행 후 실제로 본 결과가 기대와 같으면
통과에 체크, 다르면 실패에 체크하고 메모에 무슨 일이 일어났는지 짧게 적는다.

## 핵심 기능 (손으로 수행)

### 시나리오 1: 뛰다가 멈추면 시간이 실제로 멈추는지

**준비:** 러닝 탭에서 위치 권한을 이미 허용한 상태.

**수행:**
1. 가운데 "시작" 버튼을 눌러 러닝을 시작한다.
2. 30초 정도 걷는다.
3. 화면 아래쪽 일시정지 버튼(멈춤 모양 아이콘)을 누른다.
4. 그 자리에서 20~30초 정도 가만히 있으면서 시간 숫자를 지켜본다.

**확인할 것:** 일시정지 버튼을 누르는 순간 "일시정지됨"이라는 표시가 나타나고, 시간
숫자가 그 자리에서 멈춰 더 이상 올라가지 않아야 한다.

**결과:** ☑ 통과 ☐ 실패
**메모:** 사용자 확인(2026-07-16) — 정상 동작.

---

### 시나리오 2: 멈춘 채로 이동해도 거리가 늘지 않는지

**준비:** 시나리오 1에서 일시정지 상태로 멈춰 있는 상태.

**수행:**
1. 일시정지된 채로 20~30m 정도 걸어서 이동한다(가능하면 옆 건물이나 다른 골목 쪽으로).
2. 화면 아래쪽 재개 버튼(재생 모양 아이콘)을 눌러 다시 시작한다.
3. 거리 숫자를 확인한다.

**확인할 것:** 일시정지 중에 이동한 거리는 총 거리에 더해지지 않아야 한다(멈추기 직전
숫자와 재개 직후 숫자가 거의 같아야 함 — 순간이동한 것처럼 보이면 안 됨).

**결과:** ☑ 통과 ☐ 실패
**메모:** 사용자 확인(2026-07-16) — 정상 동작.

---

### 시나리오 3: 재개 후 종료하면 요약 시간이 멈춘 만큼 짧은지

**준비:** 시나리오 2에서 재개까지 마친 상태.

**수행:**
1. 재개 후 30초 정도 더 걷는다.
2. 화면 아래쪽 버튼을 길게 눌러 종료한다.
3. 요약 화면의 시간을 확인한다.

**확인할 것:** 요약 화면에 나오는 시간은 실제로 뛴(걸은) 시간만 더한 값이어야 한다 —
일시정지했던 시간(시나리오 1~2에서 멈춰 있던 시간)은 포함되면 안 된다. 대략 "시작부터
종료까지 걸린 시계 시간 − 멈춰 있던 시간"과 비슷해야 한다(초 단위로 정확히 맞을
필요는 없고, 멈춰 있던 시간만큼 확실히 짧아야 한다).

**결과:** ☑ 통과 ☐ 실패
**메모:** 사용자 확인(2026-07-16) — 정상 동작.

---

### 시나리오 4: 화면을 끄고 주머니에 넣어도 일시정지 상태가 잠금화면에 남아있는지

**준비:** 러닝을 새로 시작하고 잠시 걷다가 일시정지 버튼을 누른 상태.

**수행:**
1. 일시정지한 채로 기기 화면을 끈다(전원 버튼 한 번, 또는 주머니에 넣기).
2. 10~20초 정도 기다린다.
3. 화면을 켜서 잠금화면을 본다(잠금을 풀지 않아도 됨).

**확인할 것:** 잠금화면에 러닝 카드가 그대로 떠 있어야 하고, 카드에 "일시정지"
표시(또는 멈춤 아이콘)가 보여야 한다. 카드에 보이는 시간 숫자는 화면을 끄기 전에
멈춘 그 숫자에서 그대로 멈춰 있어야 한다(화면을 끈 동안 시간이 흘러간 것처럼 숫자가
올라가 있으면 실패).

**결과:** ☑ 통과 ☐ 실패
**메모:** 사용자 확인(2026-07-16) — 정상 동작.

---

### 시나리오 5: 일시정지 상태에서 바로 종료해도 괜찮은지

**준비:** 러닝을 새로 시작하고 잠시 걷다가 일시정지 버튼을 누른 상태.

**수행:**
1. 일시정지된 채로(재개 누르지 않고) 화면 아래쪽 버튼을 길게 눌러 바로 종료한다.

**확인할 것:** 앱이 멈추거나 이상해지지 않고 정상적으로 요약 화면으로 넘어가야 한다.
잠금화면에 남아있던 러닝 카드도 사라져야 한다.

**결과:** ☑ 통과 ☐ 실패
**메모:** 사용자 확인(2026-07-16) — 정상 동작.

---

### 시나리오 6: 방금 저장된 기록의 시간·페이스가 실제 뛴 시간 기준인지

**준비:** 시나리오 3에서 저장한 기록.

**수행:**
1. 러닝 탭 대기 화면에서 기록 버튼(목록 아이콘)을 누른다.
2. 방금 저장된 기록을 탭해 상세 화면을 연다.

**확인할 것:** 목록과 상세 화면에 보이는 시간·페이스가 시나리오 3에서 확인한 요약
화면의 시간과 같아야 한다(일시정지했던 시간이 다시 섞여 들어가 시간이 더 길게
보이면 실패).

**결과:** ☑ 통과 ☐ 실패
**메모:** 사용자 확인(2026-07-16) — 정상 동작.

### 시나리오 7: 앱을 강제 종료해도 잠금화면 카드가 잘 정리되는지

**준비:** 러닝을 시작해서 트래킹 중인 상태(일시정지 여부는 상관없음).

**수행:**
1. 최근 사용한 앱 목록(앱 스위처)을 열어 이 앱을 위로 쓸어올려 강제 종료한다.
2. 앱을 다시 실행한다.
3. 잠금화면을 확인한다(화면을 껐다 켜서 봐도 됨).

**확인할 것:** 강제 종료 후에는 잠금화면에 남아있던 러닝 카드가 사라져 있어야 한다
(계속 떠 있으면 실패).

**결과:** ☐ 통과 ☐ 실패
**메모:** 이번 라운드에서는 확인 안 함 — 다음 실기기 QA 기회로 재이월(2026-07-16, 사용자).

### 시나리오 8: 20분 이상 뛰었을 때 배터리가 눈에 띄게 빨리 닳는 느낌인지

**준비:** 시간 여유가 있을 때 한 번, 20분 이상 실제로 걷거나 뛰어본다(일시정지도 중간에
한두 번 섞어서 사용해본다).

**수행:**
1. 시작 전 배터리 잔량을 확인한다.
2. 20분 이상 러닝(중간에 일시정지 포함)을 진행한다.
3. 종료 후 배터리 잔량을 다시 확인한다.

**확인할 것:** 다른 러닝 앱을 오래 켜뒀을 때와 비교해서 유난히 빨리 닳는다는 느낌이
없어야 한다(위치 추적 기능 특성상 어느 정도 소모되는 것은 정상).

**결과:** ☐ 통과 ☐ 실패
