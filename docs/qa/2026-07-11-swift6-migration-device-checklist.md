# Swift 6 마이그레이션 실기기 체크리스트 (2026-07-11)

> 새 기능이 아니라 "내부 동시성 설정을 바꾼 뒤에도 기존 동작이 그대로인지" 확인하는 회귀
> 체크리스트입니다. 시나리오 1·2(빠른 연속 탭 / 탭 직후 팬)가 제일 중요합니다 — 최종 브랜치
> 리뷰에서 이 두 개만 실제 실행 시점 타이밍(`Task.sleep`/취소)이라 코드만 봐서는 증명이 안
> 되는 부분으로 확인됐습니다. 시나리오 4(저장→불러오기, 예전 크래시 지점)는 최종 코드에서
> 관련 타입이 전부 불변 데이터만 다루는 무격리 구조라 코드 리뷰만으로도 이미 안전이 확인된
> 상태라, 참고 삼아 한 번 훑어보는 정도면 충분합니다.
> 플랜: `docs/superpowers/plans/2026-07-10-swift6-migration.md`

빌드/테스트/린트는 전부 통과했습니다(경고 0, 테스트 178개 전체 통과). 다만 이번 세션에서는
시뮬레이터 자동 조작 도구가 이 앱 화면에서 터치에 반응하지 않는 문제가 있어서, 아래
시나리오들을 실제 손으로 확인하지 못했습니다. 도구 문제이지 앱 버그로 의심되는 정황은
없었지만, 실제로 손으로 한 번 확인해야 안심할 수 있는 항목들입니다.

**사용법:** 위에서부터 순서대로 하나씩 수행한다. 수행 후 실제로 본 결과가 기대와 같으면
통과에 체크, 다르면 실패에 체크하고 메모에 무슨 일이 일어났는지 짧게 적는다.

## 참고: 이미 확인된 것 (다시 안 해도 됨)

- 탭 2회로 경로 생성 + 거리 표시: 자동 테스트(`testSelectingTwoPointsShowsDistance`)가 실제
  화면을 탭해서 통과 확인함.
- 더블탭 시 줌만 되고 점이 안 찍히는 것: 판별 로직 단위 테스트로 확인됨.

## 핵심 확인 (손으로 수행 — 여기가 이번에 진짜 필요한 부분)

> **결과: 전체 통과 (2026-07-11, 사용자 실기기 확인).**

### 시나리오 1: 빠른 연속 탭에서 마커·점이 겹치거나 남지 않는지

**준비:** 앱을 열고 지도가 보이는 상태.

**수행:**
1. 지도의 서로 다른 지점을 아주 빠르게 연속으로 3~4번 톡톡 탭한다(간격을 거의 안 두고).
2. 잠시(1~2초) 기다린다.
3. 화면에 남은 마커와 점의 개수를 센다.

**확인할 것:** 의도한 만큼만 점이 찍혀야 한다 — 너무 빨리 탭해서 생긴 "유령 마커"나 사라져야
할 임시 마커가 남아있으면 안 된다. 앱이 멈추거나 이상하게 버벅이지 않아야 한다.

**결과:** ☑ 통과 ☐ 실패

### 시나리오 2: 지도를 탭한 직후 바로 손가락으로 화면을 밀면(팬) 점이 안 찍히는지

**준비:** 지도가 보이는 상태.

**수행:**
1. 지도의 한 지점을 톡 탭한 직후(마커가 뜨기도 전에), 바로 이어서 그 자리에서 화면을 다른
   방향으로 쭉 밀어(드래그) 지도를 이동시킨다.
2. 지도 이동이 끝나면 손을 뗀다.

**확인할 것:** 방금 탭했던 자리에 점이 찍히면 안 된다(탭 다음에 바로 이어진 팬은 "지도를
움직이려는 의도"로 취소돼야 함). 지도는 정상적으로 밀려야 한다.

**결과:** ☑ 통과 ☐ 실패

### 시나리오 3: 그리기 모드로 손가락을 쭉 그으면 그 경로가 코스에 붙는지

**준비:** 그리기 모드로 전환하는 버튼을 누른다.

**수행:**
1. 지도 위에서 손가락을 떼지 않고 곡선으로 쭉 긋는다(스트로크).
2. 손가락을 뗀다.

**확인할 것:** 그은 궤적을 따라 경로 선이 지도에 그대로 붙어야 하고, 거리 숫자도 그만큼
늘어나야 한다. 중간에 끊기거나 엉뚱한 직선으로 이어지면 안 된다.

**결과:** ☑ 통과 ☐ 실패

### 시나리오 4: 코스를 저장했다가 목록에서 다시 불러올 때 크래시 없이 되는지 (참고용 — 재확인 정도)

> 예전에 코스를 불러올 때 지도 위 마커/경로 표시(오버레이) 처리 중 실제로 앱이 죽는 문제가
> 있었고, 그걸 고친 지점입니다. 최종 브랜치 리뷰에서 관련 타입이 전부 불변 데이터만 다뤄서
> 코드 구조상 이 종류의 크래시가 다시 날 수 없다는 게 확인됐습니다. 그래도 한 번은 눈으로
> 보고 넘어가는 정도로 수행하면 됩니다.

**준비:** 지도를 탭하거나 그려서 점이 여러 개(3~4개 이상) 있는 코스를 하나 만든다.

**수행:**
1. 저장 버튼으로 이름을 붙여 저장한다.
2. "초기화"로 지도를 비운다.
3. 목록을 열어 방금 저장한 코스를 탭해서 불러온다.
4. 지도에 마커와 경로선이 다시 나타나는 순간을 잘 지켜본다.
5. 같은 코스를 한 번 더 초기화 → 다시 불러오기를 반복해본다(2~3회).

**확인할 것:** 불러오는 순간 앱이 멈추거나 죽지 않아야 하고, 저장했던 마커·경로선이 정확히
그대로 다시 나타나야 한다. 반복해서 불러와도 매번 정상이어야 한다.

**결과:** ☑ 통과 ☐ 실패
