# MVP9 edit-consistency 실기기 체크리스트 (2026-07-04)

> 이어붙이기 규칙 교체(자동 방향 감지 제거), 끝점 근접 탭 왕복, 닫힌 코스 병합 핀, 경유점 마커, redo.
> 스펙: `docs/superpowers/specs/2026-07-03-edit-consistency-design.md`
> 플랜: `docs/superpowers/plans/2026-07-03-edit-consistency.md`
> 템플릿: `docs/agent-rules/testing.md`

시뮬레이터·자동화 테스트는 전부 통과(빌드/테스트 108건/린트 그린, 전체 브랜치 코드리뷰 완료 —
Critical/Important 코드 결함 없음). 아래는 시뮬레이터·유닛 테스트로는 검증할 수 없는 항목이다.

## 결과 요약 (2026-07-04 1차 실기기 QA)

실기기 QA에서 버그 5건 발견 → 전부 이번 세션에서 근본 원인 진단 후 수정·커밋 완료:
- **그리기 모드 반전(prepend)이 항상 안 되고 도착점만 움직임** — 근본 원인: 손으로 그린 스트로크의 각 구간을 실제 도로망에 라우팅(MKDirections)하는 과정에서 스냅 오차가 발생해, 출발점 바로 위에서 시작해도 결과 좌표가 20m 임계값을 넘어버려 반전 판정이 거의 항상 실패했음. 탭 모드는 애초에 기존 끝점에서 라우팅을 시작해 이 문제가 없었음. 라우팅 전에 원본 시작점이 기존 끝점 근처면 정확한 좌표로 치환하도록 수정(commit `b5e130e`).
- **핀 탭 시 MapKit 기본 선택 애니메이션(확대+흔들림)** — 커스텀 탭 처리와 무관한 MapKit 네이티브 동작이라 비활성화(commit `69d1a9f`).
- **병합 핀 체크무늬 배지가 안 보이거나 위치가 이상함** — `view.bounds` 기반 위치 계산이 레이아웃 완료 전에 실행돼 배지가 잘못된 위치에 그려지던 문제. Auto Layout 제약으로 교체(commit `69d1a9f`).
- **경유점이 거리 라벨을 랜덤하게 가림** — 두 annotation 타입 사이에 그리기 순서(z-order)가 지정 안 돼 있던 문제. 거리 라벨이 항상 위에 오도록 `zPosition` 고정(commit `69d1a9f`).
- **탭모드에서 더블탭 줌 시 핀도 같이 찍힘** — 네이티브 더블탭 줌 제스처와 싱글탭 좌표추가 제스처가 경쟁 관계로 설정 안 돼 있던 문제. `require(toFail:)`로 해소(commit `69d1a9f`).

아래 체크리스트는 이 수정들이 실기기에서 실제로 해소됐는지 재확인하는 항목과, 아직 확인 안 된 나머지 항목으로 구성된다.

## 빌드/설치

- [ ] 기기 연결, 자동 서명 팀 확인, Xcode Run 성공
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰(설정 > 일반 > VPN 및 기기 관리)

## 핵심 기능 — attach 규칙 (Task 1)

- [ ] 탭 모드에서 A→B→C 순서로 이어 그리면 정상적으로 누적되는지 (기존 동작 회귀 없음)
- [ ] **[재확인] 그리기 모드에서 출발점 근처에서 시작해 바깥쪽으로 그리면(뒤로 연장) 코스 시작이 새로 그린 쪽으로 바뀌는지** — 1차 QA에서 실패 확인, `b5e130e`로 수정 완료. 재확인 필요
- [ ] 도착점 근처에서 시작해 출발점 쪽으로 되짚어 그리면(왕복) 반전되지 않고 그대로 append되는지 — 1차 QA에서 이 항목 자체는 문제 없었음(도착점이 옮겨가는 동작은 정상)
- [ ] 코스와 전혀 동떨어진 곳에서 새 구간을 그리면 도착점에서 자동 라우팅(gap)으로 연결되고 그린 방향 그대로 붙는지

## 핵심 기능 — 출발핀 탭 왕복 (Task 4, 5)

- [x] 열린 코스 상태에서 지도 하단(statusPanel)에 "출발핀을 탭하면 왕복 완성" 힌트가 보이는지 — 1차 QA 확인됨(관련 질문 없었음)
- [x] 출발핀을 탭하면 도착점에서 출발점까지 자동으로 라우팅되어 왕복이 완성되는지 — 1차 QA에서 동작 확인(핀 스타일 혼동은 있었으나 왕복 완성 자체는 정상)
- [ ] 도착핀을 탭하면 아무 일도 일어나지 않고 "이미 도착점입니다" 안내가 뜨는지
- [ ] 왕복 완성 후(닫힌 코스) 병합 핀을 탭하면 "이미 닫힌 코스입니다" 안내가 뜨는지
- [ ] **24pt 히트 반경이 실제 손가락 터치에서 체감상 적절한지** — 시뮬레이터로 검증 불가, 실기기 전용 항목
- [ ] 지도를 확대/축소한 상태에서도 핀 히트 판정이 일관되게 느껴지는지

## 핵심 기능 — 닫힌 코스 병합 핀 (Task 6)

- [ ] **[재확인] 왕복을 완성하면 출발/도착 핀이 하나로 합쳐지고 체크무늬 배지가 붙는지** — 1차 QA에서 배지가 잘 안 보여 "출발 핀과 똑같다"고 느껴짐, `69d1a9f`로 Auto Layout 기반 위치 수정 완료. 재확인 필요
- [ ] **라우팅 스냅 오차로 인해 왕복을 완성했는데도 병합 핀이 안 뜨고 출발/도착 핀이 분리된 채로 남는 경우가 있는지** — 코드리뷰 Important 발견, 아직 실기기 재현 안 됨(1차 QA에서는 병합 자체는 성공). 지속 관찰 필요

## 핵심 기능 — 경유점 마커 (Task 7)

- [ ] 구간을 3개 이상 이어 붙였을 때 각 구간 경계마다 작은 흰 점이 보이는지
- [ ] **[재확인] 경유점이 거리 라벨을 가리지 않는지 (항상 라벨이 위)** — 1차 QA에서 "거리 라벨이 랜덤하게 안 보인다"고 확인된 z-order 버그, `69d1a9f`로 zPosition 고정 완료. 재확인 필요

## 핵심 기능 — redo (Task 2, 3)

- [ ] 되돌리기(undo) 후 "앞으로" 버튼이 활성화되고, 누르면 원래 상태로 복원되는지
- [ ] prepend로 붙은 구간을 undo→redo했을 때 같은 자리(맨 앞)로 정확히 복원되는지
- [ ] undo→redo 후 새 구간을 추가하면 "앞으로" 버튼이 다시 비활성화되는지 (미래 무효화)

## 제스처/입력 감

- [ ] 핀 탭과 일반 지도 탭(코스 연장)이 서로 헷갈리지 않고 의도대로 구분되는지
- [ ] **[재확인] 탭모드에서 화면을 빠르게 두 번 탭(더블탭 줌)해도 핀이 같이 찍히지 않는지** — 1차 QA에서 발견된 제스처 충돌, `69d1a9f`로 `require(toFail:)` 적용 완료. 재확인 필요 — 코드리뷰에서 "MapKit 내장 더블탭 인식기가 그 시점에 실제로 존재하는지"는 런타임 사실이라 실기기 확인 전에는 100% 보장 못 한다고 지적됨
- [ ] 핀 선택 애니메이션(확대+흔들림)이 더 이상 안 나오는지 — `69d1a9f`로 `isEnabled = false` 적용

## 지역/환경 의존

- [ ] 한국 도로망에서 gap 라우팅 스냅 오차가 20m 임계값과 실사용상 잘 맞는지

## 엣지/견고성

- [ ] 매우 짧은 왕복(예: 몇 미터 이내)에서도 닫힌 코스로 정상 판정되는지
- [ ] 구간이 매우 많이 쌓인 상태(10개 이상)에서 redo 다단계가 순서대로 정확히 복원되는지

## 의도 일치

- [x] 만든 기능이 원한 UX와 맞는가 — 1차 QA에서 상세히 다뤄짐. 버그 5건 발견·수정, 나머지는 의도대로 동작
