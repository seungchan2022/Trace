# MVP9 edit-consistency 실기기 체크리스트 (2026-07-04)

> 이어붙이기 규칙 교체(자동 방향 감지 제거), 끝점 근접 탭 왕복, 닫힌 코스 병합 핀, 경유점 마커, redo.
> 스펙: `docs/superpowers/specs/2026-07-03-edit-consistency-design.md`
> 플랜: `docs/superpowers/plans/2026-07-03-edit-consistency.md`
> 템플릿: `docs/agent-rules/testing.md`

시뮬레이터·자동화 테스트는 전부 통과(빌드/테스트 108건/린트 그린, 전체 브랜치 코드리뷰 완료 —
Critical/Important 코드 결함 없음). 아래는 시뮬레이터·유닛 테스트로는 검증할 수 없는 항목이다.

## 빌드/설치

- [ ] 기기 연결, 자동 서명 팀 확인, Xcode Run 성공
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰(설정 > 일반 > VPN 및 기기 관리)

## 핵심 기능 — attach 규칙 (Task 1)

- [ ] 탭 모드에서 A→B→C 순서로 이어 그리면 정상적으로 누적되는지 (기존 동작 회귀 없음)
- [ ] 그리기 모드에서 출발점 근처에서 시작해 바깥쪽으로 그리면(뒤로 연장) 코스 시작이 새로 그린 쪽으로 바뀌는지 (반전 prepend, 유일한 반전 케이스)
- [ ] 도착점 근처에서 시작해 출발점 쪽으로 되짚어 그리면(왕복) 반전되지 않고 그대로 append되는지 — **이전 버전에서 핀 라벨이 뒤집히던 버그의 핵심 수정 지점**
- [ ] 코스와 전혀 동떨어진 곳에서 새 구간을 그리면 도착점에서 자동 라우팅(gap)으로 연결되고 그린 방향 그대로 붙는지

## 핵심 기능 — 출발핀 탭 왕복 (Task 4, 5)

- [ ] 열린 코스 상태에서 지도 하단(statusPanel)에 "출발핀을 탭하면 왕복 완성" 힌트가 보이는지
- [ ] 출발핀을 탭하면 도착점에서 출발점까지 자동으로 라우팅되어 왕복이 완성되는지 (탭 좌표가 아니라 출발점 좌표로 스냅되는지)
- [ ] 도착핀을 탭하면 아무 일도 일어나지 않고 "이미 도착점입니다" 안내가 뜨는지
- [ ] 왕복 완성 후(닫힌 코스) 병합 핀을 탭하면 "이미 닫힌 코스입니다" 안내가 뜨는지
- [ ] **24pt 히트 반경이 실제 손가락 터치에서 체감상 적절한지** (너무 좁아서 안 잡히거나, 너무 넓어서 의도치 않게 핀에 잡히는지) — 시뮬레이터로 검증 불가, 실기기 전용 항목
- [ ] 지도를 확대/축소한 상태에서도 핀 히트 판정이 일관되게 느껴지는지 (화면 포인트 기준이라 줌과 무관해야 함)

## 핵심 기능 — 닫힌 코스 병합 핀 (Task 6)

- [ ] 왕복을 완성하면 출발/도착 핀이 하나로 합쳐지고 체크무늬 배지가 붙는지
- [ ] **병합 핀 배지 위치가 핀 아이콘 오른쪽 위에 정확히 붙어 보이는지, 혹시 첫 렌더링 시 위치가 어긋났다가 다시 그려지는지** — 코드리뷰에서 `view.bounds` 레이아웃 타이밍 이슈 가능성이 지적된 항목, 실기기 확인 필요
- [ ] **라우팅 스냅 오차로 인해 왕복을 완성했는데도 병합 핀이 안 뜨고 출발/도착 핀이 분리된 채로 남는 경우가 있는지** — 코드리뷰 Important 발견: 유닛 테스트는 정확한 좌표 스텁이라 이 케이스를 못 잡음, 실제 도로망 라우팅에서 20m 이상 벗어나는 스냅이 발생하면 재현 가능

## 핵심 기능 — 경유점 마커 (Task 7)

- [ ] 구간을 3개 이상 이어 붙였을 때 각 구간 경계마다 작은 흰 점이 보이는지
- [ ] 경유점이 출발/도착/병합 핀과 겹칠 때 항상 핀이 우선하고 경유점이 가려지는지 (조용한 위계)

## 핵심 기능 — redo (Task 2, 3)

- [ ] 되돌리기(undo) 후 "앞으로" 버튼이 활성화되고, 누르면 원래 상태로 복원되는지
- [ ] prepend로 붙은 구간을 undo→redo했을 때 같은 자리(맨 앞)로 정확히 복원되는지
- [ ] undo→redo 후 새 구간을 추가하면 "앞으로" 버튼이 다시 비활성화되는지 (미래 무효화)

## 제스처/입력 감

- [ ] 핀 탭과 일반 지도 탭(코스 연장)이 서로 헷갈리지 않고 의도대로 구분되는지
- [ ] 빠른 연속 탭에서 핀 히트 판정이 씹히거나 중복 실행되지 않는지

## 지역/환경 의존

- [ ] 한국 도로망에서 gap 라우팅 스냅 오차가 20m 임계값과 실사용상 잘 맞는지 (너무 자주 반전/병합이 오작동하지 않는지)

## 엣지/견고성

- [ ] 매우 짧은 왕복(예: 몇 미터 이내)에서도 닫힌 코스로 정상 판정되는지
- [ ] 구간이 매우 많이 쌓인 상태(10개 이상)에서 redo 다단계가 순서대로 정확히 복원되는지

## 의도 일치

- [ ] 만든 기능이 원한 UX와 맞는가 (불일치 메모)

## 알려진 한계 (버그 아님)

- [ ] 이번 개발 세션 내(에이전트 작업 중)에는 지도 탭 제스처 자동화 도구 한계로 시뮬레이터 라이브 확인이 안 돼 대부분 코드 리뷰 + 유닛 테스트로만 검증됨 — 전체 브랜치 코드리뷰(opus)에서 attach/redo/annotation 3계열의 교차 통합 지점을 직접 코드 추적으로 검증했으나 실기기 동작 확인은 아님
