# 러닝 기록 저장 실기기 체크리스트 (2026-07-14)

> 러닝을 마치면 기록이 자동으로 저장되고, 러닝 탭에서 지난 기록을 목록으로 보고
> 지울 수 있는 새 기능입니다. 뛸 필요는 없습니다 — 짧게 한 번(1~2분) 걷거나 제자리에서
> 움직이는 정도로 충분합니다. 시뮬레이터로는 "앱을 완전히 껐다 켜도 기록이 정말
> 남아 있는지"와 "밀어서 지우는 손동작"을 확인할 수 없어서 아래 항목들을 실제
> 기기에서 봐주시면 좋겠습니다.

빌드/테스트는 전부 통과했습니다(자동 테스트 전체 통과, 회귀 없음). 시뮬레이터에서는
저장→목록→상세→(스와이프 삭제 제외) 흐름을 눈으로 확인했고 정상이었습니다.

**사용법:** 위에서부터 순서대로 하나씩 수행한다. 수행 후 실제로 본 결과가 기대와 같으면
통과에 체크, 다르면 실패에 체크하고 메모에 무슨 일이 일어났는지 짧게 적는다.

## 빌드/설치
- [ ] 기기 연결, 자동 서명 팀 확인, Xcode Run 성공
- [ ] 첫 실행 시 기기에서 개발자 인증서 신뢰(설정 > 일반 > VPN 및 기기 관리)

## 핵심 기능 (손으로 수행)

### 시나리오 1: 산책을 마치면 "기록 저장됨"이 보이는지

**준비:** 러닝 탭에서 위치 권한을 이미 허용한 상태.

**수행:**
1. 가운데 "시작" 버튼을 눌러 기록을 시작한다.
2. 1~2분 정도 걷거나 제자리에서 몸을 움직인다.
3. 아래쪽 버튼을 길게 눌러 종료한다.
4. 종료 후 뜨는 요약 화면을 본다.

**확인할 것:** 요약 화면 위쪽에 "기록 저장됨"이라는 문구가 체크 표시와 함께 나와야
한다. 만약 잠깐 "저장 중…"이 보였다가 "기록 저장됨"으로 바뀌어도 정상이다. 만약
"저장 실패"와 "다시 시도" 버튼이 보이면, "다시 시도"를 한 번 눌러서 "기록 저장됨"으로
바뀌는지도 같이 확인한다.

**결과:** ☐ 통과 ☐ 실패

### 시나리오 2: 기록 버튼을 누르면 방금 산책이 목록 맨 위에 있는지

**준비:** 시나리오 1을 마치고 요약 화면에서 "닫기"를 누른 상태.

**수행:**
1. 러닝 탭 대기 화면 오른쪽 위의 기록 버튼(목록 아이콘)을 누른다.

**확인할 것:** 목록 맨 위에 방금 걸었던 기록이 나와야 한다. 날짜/시간, 거리, 걸린
시간, 페이스가 같이 보여야 하고, 거리·시간이 0이 아니어야 한다(짧게 걸었어도 값이
있어야 함).

**결과:** ☐ 통과 ☐ 실패

### 시나리오 3: 기록을 누르면 지도에 지나온 길이 보이는지

**준비:** 시나리오 2에서 연 목록 화면.

**수행:**
1. 방금 확인한 기록을 탭한다.

**확인할 것:** 상세 화면으로 넘어가고, 위쪽 지도에 걸었던 길이 선으로 표시되어야
한다(아주 짧은 거리라 선이 작게 보여도 정상). 아래쪽에는 거리·시간·평균 페이스·고도
상승 네 개 숫자가 보여야 한다.

**결과:** ☐ 통과 ☐ 실패

### 시나리오 4: 앱을 완전히 껐다 켜도 기록이 남아 있는지

**준비:** 시나리오 3까지 마친 상태.

**수행:**
1. 상세 화면에서 뒤로 나가고, 목록 화면도 닫는다.
2. 최근 사용한 앱 목록(앱 스위처)을 열어 이 앱을 위로 쓸어올려 완전히 종료한다.
3. 이 앱을 다시 실행한다.
4. 러닝 탭에서 기록 버튼을 눌러 목록을 다시 연다.

**확인할 것:** 앱을 완전히 껐다 켰는데도 아까 확인했던 기록이 목록에 그대로 남아
있어야 한다("아직 기록이 없어요"로 비어 보이면 실패).

**결과:** ☐ 통과 ☐ 실패

### 시나리오 5: 기록을 밀어서 지우면 사라지고, 다시 켜도 없는지

**준비:** 시나리오 4를 마치고 목록에 기록이 남아 있는 상태.

**수행:**
1. 목록에서 기록을 손가락으로 왼쪽으로 민다.
2. 나타나는 삭제 버튼을 누른다.
3. "삭제할까요?" 확인 창이 뜨면 "삭제"를 누른다.
4. 목록에서 사라졌는지 본다.
5. 앱을 완전히 껐다가 다시 켜서 기록 버튼으로 목록을 다시 연다.

**확인할 것:** 3번 이후 목록에서 그 기록이 바로 사라져야 한다. 5번처럼 앱을 완전히
껐다 켜도 지운 기록이 다시 나타나지 않아야 한다(진짜로 지워졌는지 확인하는
단계이다).

**결과:** ☐ 통과 ☐ 실패

## 엣지/견고성

### 시나리오 6: 기록이 하나도 없을 때 목록 화면이 어떻게 보이는지

**준비:** 아직 한 번도 러닝을 저장한 적이 없거나, 시나리오 5로 마지막 기록을 지운
직후.

**수행:**
1. 러닝 탭 대기 화면에서 기록 버튼을 누른다.

**확인할 것:** 목록이 텅 빈 화면 대신 "아직 기록이 없어요"라는 안내 문구와 함께
"러닝을 마치면 기록이 자동으로 저장돼요" 같은 설명이 보여야 한다(빈 화면만 덩그러니
있으면 실패).

**결과:** ☐ 통과 ☐ 실패
