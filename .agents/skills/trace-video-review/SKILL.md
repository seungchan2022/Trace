# 외부 영상/콘텐츠 리뷰

> Codex에서는 `$trace-video-review`, Claude Code에서는 `/trace-video-review`로 호출한다.
> 목적: 유튜브 등에서 발견한 AI/개발 워크플로우 팁이 Trace 작업 방식에 적용할 만한지 판단한다.
> 이 스킬은 로드맵·마일스톤·플랜 진행 상태를 다루지 않는다. 필요하면 Codex `$trace-init` 또는 Claude Code `/trace-init`을 별도로 호출한다.

## 전제

- 사용자는 유튜브 링크 또는 영상 속 프롬프트/텍스트를 직접 붙여넣는다. 링크만 왔으면 로컬에
  설치된 `yt-dlp`로 **원본 언어** 자막을 추출해서 읽는다 — 자동 번역 자막은 쓰지 않는다(ASR
  오류 위에 번역 오류가 겹쳐 부정확도가 커진다). 텍스트/프롬프트를 이미 붙여넣었으면 다시
  추출하지 않는다.

## 수행 절차

1. **내용 파악** — 영상/텍스트가 제안하는 기법·프롬프트·워크플로우를 정리한다.
2. **기존 상태 대조** — `docs/agent-rules/*`와 관련 메모리를 먼저 확인해, 이 기법이 이미 어느
   정도 구현돼 있는지 짚는다. "이미 갖고 있다"는 확인이 새로 도입할 목록보다 먼저다.
3. **델타만 추출** — 진짜 새로운 부분만 추린다. 특히:
   - 모델/도구 선택(`/model`, 어떤 모델을 메인/서브에이전트로 쓸지 등)은 사용자의 개인 설정이지
     레포 규칙이 아니다 — 레포 규칙 문서에 넣지 않는다.
   - `advisor` 도구처럼 "언제 검토를 거칠지" 같은 에이전트 행동은 레포 규칙 후보가 될 수 있다.
   - 새 외부 도구(설치형) 도입은 보안·과설치 여부를 먼저 따진 뒤에만 제안한다.
4. **판단 제시** — `docs/agent-rules/skills.md`의 "Asking the User Decisions" 규칙대로 채팅에
   plain text A/B/C로 제안하고 추천을 표시한다. 결정은 사용자 몫이다.
5. **반영** — 승인되면 해당 `docs/agent-rules/*.md`에 반영한다. 커밋 전에는 `git status`/`git
   diff`로 다른 세션의 미커밋 변경과 섞이지 않는지 확인하고, 필요한 파일·hunk만 콕 집어
   스테이징한다.
